CPSR Agent System — 한 장 정리

사내 CPSR 자동작성 AI 에이전트, acceptance 95%. 6개 레슨이 도달한 모든 결론을 한 페이지로.

1. Mission

처방·노출·독성 자료를 통합 추론해 CPSR Annex I Part A + Part B 초안을 LLM 에이전트가 생성. 사내 인증 SA가 거의 손대지 않고 서명·검토 가능한 수준. SCCS NoG 12th Rev (2023) 1차 기준, leave-on 스킨케어 우선, 솔로 메이커.

3. 핵심 수식 (Lesson 0003 · Tier 1)

SED = A × C × F × DAp / BW   [mg/kg bw/day]
MoS = NOAEL / SED ≥ 100
기호값/단위비고
Ag/daySCCS Table 1 — face cream 1.54
Cfraction% ÷ 100 (4% → 0.04)
F0~1Leave-on 1.0, Rinse-off ≪1
DApfraction측정값 없으면 0.5 default
BW60 kgSCCS default
NOAELmg/kg bw/dayTier 3에서 추출, 보수성+품질 가중
MoS≥ 100종간차 10 × 개체차 10 (default UF)
단위 함정 5종: % vs fraction · g vs mg · retention 0 vs 1 · DA% vs DA fraction · 종 NOAEL을 인간에 그대로 적용 금지.
Aggregate: SEDagg = Σ SEDi (동일 성분, 여러 제품 동시 사용).

4. 5-Tier 자동화 분류 (Lesson 0002 · Backbone)

TIER 1Deterministic Compute
예:
SED·MoS·retention·단위 변환·aggregate 합산
구현:
순수 함수, unit test 100%. LLM 금지.
실패:
입력 단위 혼동, 농도 합 ≠ 100%
TIER 2Structured Data Retrieval
예:
CosIng INCI 정규화·Annex 상태·CAS lookup·내부 R&D DB
구현:
API client + Substance Identity Hub. CosIng API 없음.
실패:
INCI 변종, 동의어, 한국명, 염·수화물, 복합 성분
TIER 3LLM Extraction with Grounding (RAG)
예:
SCCS Opinion·CIR·ECHA PDF에서 NOAEL/DA% 추출
구현:
Pydantic 스키마 + Citation 강제 + evaluator (quote literal match)
실패:
fabrication, 종 오인, 단위 혼동, 보수값 미선택
TIER 4LLM Reasoning with Rubric Evaluation
예:
B.3 Reasoning, B.1 Conclusion, A.4·A.8 narrative
구현:
Generator → Evaluator(rubric) → Refiner 루프. 3회 retry 후 HITL.
실패:
boilerplate, 갭 침묵, 민감군 미고려, MoS<100인데 safe 결론
TIER 5 / HITLHuman-in-the-Loop (필수)
예:
B.4 자격자 서명, MoS<100, 신성분, Evaluator retry 실패
구현:
워크플로우 게이트 + 알림 + 결정 audit log → 학습 데이터
실패:
silent failure → HITL 호출 누락 (false negative)

5. 시스템 아키텍처 (Lesson 0002)

INPUT: {formulation_INCI%, product_category, use_scenario} | v [T1] formulation_validator # sum=100%, units, splits [T2] cosing_normalizer # INCI → substance_id + CAS + Annex [T1] exposure_calculator # A·C·F·DA·BW → SED per substance [T2] internal_data_fetcher # R&D, stability, microbio | v [T3] tox_rag # SCCS Opinion + CIR + ECHA → NOAEL └─ evaluator # quote literal + unit + species check | v [T1] mos_calculator # MoS = NOAEL/SED, flag <100 | v [T4] reasoning_generator # B.3 per-substance + global └─ evaluator(rubric) # R1.1~R2.6 [T4] conclusion_generator # B.1 + B.2 | v [T5] HITL gate # SA review + sign B.4 | v OUTPUT: CPSR draft (JSON) + Word/PDF + audit log

6. Tier 2 — Substance Identity Hub (Lesson 0005)

표기예 (Niacinamide)
INCI canonicalNiacinamide
CAS98-92-0
EC202-713-4
한국명나이아신아마이드
IUPACPyridine-3-carboxamide
동의어Nicotinamide, Vitamin B3

중심은 substance_id (내부 UUID). INCI는 변할 수 있어도 substance_id는 immutable. 모든 NOAEL·MoS·과거 CPSR 이력이 이 키로 묶임.

해결 chain: exact → synonym → fuzzy (≥0.95) → LLM propose (0.7, HITL).

처방 입력 검증 6단계: ① substance_id 해결 ② 농도 합 99.5~100.5% ③ (and)/+ 분할 ④ Annex II 금지 ⑤ Annex III/V/VI 한도 ⑥ 알레르겐 26+24 합산.

7. Tier 3 — SCCS Opinion 해부 (Lesson 0004)

섹션내용
3.1.1CAS, EC, INCI — Tier 2 매칭
3.3.4Dermal absorption → Tier 1 DAp
3.3.5⭐⭐ Repeated dose — NOAEL 본거지
3.3.13⭐⭐⭐ Safety eval 표 — NOAEL+SED+MoS 한 표에
4⭐⭐⭐ Conclusion verdict — 우리 B.1 모델

추출 4규칙: ① "Cite or refuse" ② "Choose conservative" (extract all → deterministic min) ③ "Species and route always" ④ "Units mg/kg bw/day".

Fallback chain: SCCS → CIR → ECHA → EFSA → HITL. confidence 등급 동반.
