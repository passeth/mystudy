STORM / Co-STORM 파이프라인 맵

한 장으로 흐름을 복기할 때 쓴다. 상세 설명은 레슨으로.

STORM

Topic
  │
  ├─ Pre-writing
  │    ├─ Discover perspectives (유사 문서 스캔)
  │    ├─ Simulated multi-perspective Q&A (검색 그라운딩)
  │    └─ Curate → Outline
  │
  └─ Writing
       ├─ Generate article sections with citations
       └─ Polish (summary / dedupe)

Runner 제어 스위치

  • do_research — 관점·대화·자료 수집
  • do_generate_outline — 개요
  • do_generate_article — 본문
  • do_polish_article — 다듬기

Co-STORM

Topic
  │
  ├─ warm_start()  → 공유 개념 공간 시드
  │
  ├─ discourse loop
  │    ├─ Expert agents (근거 답 / 후속 질문)
  │    ├─ Moderator (미사용 정보 자극 질문)
  │    └─ Human (관찰 또는 발화 주입)
  │         └─ dynamic mind map update
  │
  └─ reorganize knowledge base → generate_report()

PDP 기획에 얹는 최소 매핑

Perspectives
타깃 고민, 경쟁, 성분, 루틴, 클레임 리스크
Retriever corpus
자사 자료, 경쟁 PDP, 후기, 규제/성분 DB
Outline
상세페이지 섹션 스토리라인
Human gate
효능 표현·금지 문구 검수 (필수)
STORM 출력은 출판/광고 최종본이 아니다. 조사·개요·초안 레이어로 취급하고, 도메인 가드레일을 바깥에 둔다.
