Git/GitHub 빠른 용어표

헷갈릴 때 이 표만 다시 보면 됩니다. “내 컴퓨터”, “GitHub branch”, “PR 검토 공간”, “main 공식 버전”을 구분하는 것이 핵심입니다.

Git 기본

단어한 줄 설명헷갈리지 말 것
commit내 변경 묶음을 이력으로 저장push가 아니다. commit만으로 GitHub에 올라가지 않는다.
branchmain과 분리된 작업 줄기작업 branch가 있어도 main은 아직 안 바뀐다.
push내 branch/commit을 GitHub에 올림merge가 아니다. 올렸을 뿐이다.
pullGitHub의 변경을 내 컴퓨터로 가져와 반영push의 반대 방향이다.
PRbranch 변경을 main에 넣어도 되는지 검토 요청push와 다르다. 리뷰 공간이다.
merge승인된 branch를 main에 합침보통 PR gate 통과 후 한다.

GitHub 메뉴

메뉴무엇을 보는 곳인가초보자 질문
Code파일, branch, commit, README현재 공식 코드가 무엇인가?
Issues해야 할 일과 논의무엇을 해야 하나?
Pull requests변경 검토와 merge 요청이 변경을 넣어도 되나?
AgentsAI coding agent 작업AI가 어떤 branch/PR을 만들었나?
Actions자동 테스트/빌드/배포 로그왜 check가 실패했나?
Projects작업 관리판일감들이 어떤 상태인가?
Wiki장기 운영 문서팀 규칙과 사용법은 어디 있나?
Security and quality보안/품질 경고위험한 secret이나 취약점이 있나?
Insights활동 통계최근 프로젝트 활동이 어떤가?
Settings저장소 운영 설정권한과 보호 규칙은 어떻게 되어 있나?
