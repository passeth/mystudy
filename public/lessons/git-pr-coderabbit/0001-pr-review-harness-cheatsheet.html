Reference 0001

PR 리뷰 하네스 치트시트

merge blocker를 추측하지 않고 Git, GitHub, Actions, CodeRabbit 신호로 분리해 진단하는 빠른 참조표입니다.

핵심 모델: 4개의 신호를 분리한다

  1. GitHub review state: 사람 리뷰의 Comment / Approve / Request changes. CodeRabbit check 성공과 같지 않다.
  2. Status checks: CI, CodeRabbit commit status, security scanner, required check의 success/failure/pending/skipped.
  3. Ruleset / branch protection: required reviews, required status checks, conversation resolution, linear history, merge queue 등 정책.
  4. CodeRabbit robot-review evidence: walkthrough, blocking findings, warnings, pre-merge check, current-head SHA 기준의 봇 증거.

증거 수집 명령

# 1) Git 설정 출처 확인
git config --list --show-origin

git config --show-origin user.name
git config --show-origin user.email

# 2) PR 핵심 상태 확인
gh pr view <PR_NUMBER_OR_URL>   --json number,headRefOid,mergeable,mergeStateStatus,reviewDecision,statusCheckRollup,latestReviews

# 3) required check만 보기
gh pr checks <PR_NUMBER_OR_URL> --required

# 4) 리뷰 객체와 check-run/status를 분리해서 보기
gh api repos/<OWNER>/<REPO>/pulls/<PR>/reviews
gh api repos/<OWNER>/<REPO>/commits/<HEAD_SHA>/status
gh api repos/<OWNER>/<REPO>/commits/<HEAD_SHA>/check-runs

# 5) ruleset/branch protection 확인
gh api repos/<OWNER>/<REPO>/rulesets --jq '.[] | {name, enforcement, conditions, rules}'

CodeRabbit commands

  • @coderabbitai review: incremental review
  • @coderabbitai full review: 전체 PR 재리뷰
  • @coderabbitai pause / resume: 자동 리뷰 일시정지/재개
  • @coderabbitai resolve: CodeRabbit 이전 코멘트 resolve 요청
  • @coderabbitai approve: 설정이 켜져 있을 때만 approval 시도
  • @coderabbitai run pre-merge checks: pre-merge checks 재실행

위험한 오진

  • CodeRabbit check success를 GitHub APPROVED 리뷰로 착각하기.
  • stale head SHA의 리뷰/스크린샷을 current-head evidence로 착각하기.
  • 보안 scanner 실패를 continue-on-error로 숨기기.
  • pull_request_target에서 PR branch 코드를 checkout/실행하기.

판단 알고리즘

  1. PR head SHA를 먼저 적는다.
  2. GitHub reviewDecision이 막는지 본다.
  3. required status checks가 막는지 본다.
  4. ruleset/branch protection이 요구하는 계약을 읽는다.
  5. CodeRabbit findings가 current-head에서 해결/반박/무시 처리되었는지 본다.
  6. 수정 후 같은 명령을 다시 실행해 before/after evidence를 남긴다.

초보자 판단 기준

먼저 reviewDecision, statusCheckRollup, branch protection을 따로 봅니다. 한 화면에서 같이 보여도 서로 다른 gate입니다.

관련 lesson

PR, Actions, CodeRabbit 운영 루프에서 같은 내용을 실제 작업 흐름으로 다시 볼 수 있습니다.
