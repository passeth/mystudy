GitHub beginner course

Git, PR, CodeRabbit, GitHub 상단 메뉴 입문

RISELAB 화면을 기준으로 Git의 기초 단어와 GitHub 메뉴를 연결해서 이해하는 짧은 코스입니다. 목표는 “어디를 눌러야 하는지”와 “무엇을 보고 판단해야 하는지”를 분리해서 아는 것입니다.

가장 중요한 그림

수정내 컴퓨터에서 파일을 바꿉니다.
commit변경 묶음에 이름표를 붙입니다.
branch안전한 작업 줄기에서 실험합니다.
push내 branch를 GitHub로 올립니다.
PRmain에 넣어도 되는지 요청합니다.
merge승인된 변경을 main에 합칩니다.

학습 순서

Lesson 0003: Git 기초

commit, branch, push, pull, PR, merge를 초보자 언어로 정리합니다.

Lesson 0004: GitHub 상단 메뉴

Code, Pull requests, Agents, Actions, Projects, Wiki, Security, Insights, Settings를 하나씩 봅니다.

Lesson 0005: PR 운영 루프

PR이 열리고 Actions, CodeRabbit, 사람 approval을 거쳐 merge되는 흐름을 익힙니다.

Reference: 용어표

자주 헷갈리는 단어를 한 장짜리 표로 다시 봅니다.

화면 상단 메뉴 빠른 지도

Code Issues Pull requests Agents Actions Projects Wiki Security and quality Insights Settings

작업할 때

Code에서 파일과 commit을 봅니다. 내 변경은 branch에서 만들고 push합니다.

넣어도 되는지 볼 때

Pull requests와 Actions에서 리뷰, 테스트, 실패 로그를 봅니다.

운영 규칙을 바꿀 때

Settings에서 branch protection, Actions 권한, collaborator, secrets를 봅니다. 조심해야 하는 영역입니다.
