RISELAB PR 게이트 빠른 참조

RISELAB에서 변경을 만들고, PR을 열고, CodeRabbit과 GitHub Actions 결과를 읽고, 승인 후 merge하는 최소 운영 절차입니다.

한 줄 모델

작업자는 branch에서 고치고, GitHub Actions는 기계 검증을 하고, CodeRabbit은 리뷰 의견을 주고, 사람 reviewer가 승인해야 main으로 들어간다.

현재 RISELAB 규칙

규칙의미근거 위치

한국어 PRPR 제목, 본문, 리뷰 답변, merge note는 한국어로 쓴다..github/workflows/pr-checks.yml
필수 체크Minimal Verification이 통과해야 merge 가능하다..github/workflows/pr-checks.yml

대화 해결리뷰 thread가 unresolved이면 merge가 막힌다.GitHub branch protection

상태 읽기

보이는 값뜻다음 행동
Minimal Verification: SUCCESS테스트와 Agentation build가 통과했다.리뷰/대화 상태를 본다.
Minimal Verification: FAILURE테스트, build, PR 한국어 검증 중 하나가 실패했다.로그 comment를 읽고 같은 branch에 수정 commit을 push한다.
CodeRabbit: SUCCESSCodeRabbit status가 통과했다.GitHub 승인 리뷰와 동일시하지 않는다.
reviewDecision: REVIEW_REQUIRED아직 사람 approval이 부족하다.reviewer에게 승인 요청한다.
mergeStateStatus: BLOCKED체크, 승인, unresolved thread 중 하나가 merge를 막고 있다.세부 blocker를 분리해서 확인한다.

다른 프로젝트에 옮기는 체크리스트

  1. AGENTS.md에 GitHub Flow, PR 필수, 한국어 PR 여부, 자동 self-fix loop를 적는다.
  2. .github/workflows/pr-checks.yml을 만들고 그 프로젝트의 최소 검증 명령으로 바꾼다.
  3. .github/pull_request_template.md를 만든다.
  4. CodeRabbit 또는 robot reviewer 대응 스킬을 .agents/skills/에 둔다.
  5. GitHub settings에서 main branch protection을 켠다.
  6. 필수 status check 이름이 workflow job 이름과 정확히 맞는지 확인한다.
  7. 테스트용 PR을 하나 열어 실패/성공/승인 대기를 실제로 확인한다.

기억할 점

CodeRabbit 통과, Actions 통과, 사람 approval은 서로 다른 조건입니다. merge가 막혔을 때는 하나로 뭉뚱그리지 말고 각각 분리해서 확인합니다.
