Lesson 4 / 5

Cross-linking과 지식 그래프

개념 간 연결로 트리를 넘어 그래프 지식 구조 만들기

🌐 왜 Cross-linking인가?

디렉토리 구조는 트리(tree)를 만듭니다 — 부모/자식 관계만 표현 가능합니다. Cross-link는 그래프(graph)를 만듭니다 — 어떤 관계도 표현 가능합니다.

예: products/ceraclean-foam.md는 ingredients/niacinamide.md를 참조합니다. 이 두 개념은 디렉토리상 형제가 아니지만, 링크로 연결됩니다.

파란 화살표: products/ceraclean → ingredients/niacinamide 크로스링크

🔗 두 가지 링크 형식

✅ 절대 링크 (권장)
[나이아신아마이드](/ingredients/niacinamide.md)
  • /로 시작
  • Bundle 루트 기준
  • 파일 이동해도 안정적
  • 권장 형식
⚠️ 상대 링크
[나이아신아마이드](../ingredients/niacinamide.md)
  • 상대 경로
  • 현재 파일 위치 기준
  • 파일 이동 시 깨질 수 있음
  • 필요시 사용 가능
⚠️ 깨진 링크 OK

OKF 명세: "Consumers는 깨진 링크를 허용(tolerate)해야 합니다." 링크 대상이 아직 작성되지 않은 지식을 나타낼 수 있습니다. 깨진 링크가 있어도 Bundle이 무효(malformed)가 되지 않습니다.

🏷️ 링크의 의미론

OKF 링크는 타입이 없습니다. 관계의 종류는 주변 텍스트로 전달됩니다:

관계 종류표현 방식 예시
조인 관계Joined with [customers](/tables/customers.md) on customer_id.
구성 요소주요 성분: [나이아신아마이드](/ingredients/niacinamide.md)
선행 조건이 플레이북 전에 [안정성 테스트](/processes/stability-test.md) 완료 필요.
참고/관련관련 성분: [레티놀](/ingredients/retinol.md) 참조.

링크 자체는 방향이 있는 엣지(directed edge)입니다. 그래프 뷰어는 역방향 "인용됨(Cited by)" 목록을 자동 계산합니다.

🧠 이해 확인 퀴즈

Q1. OKF에서 링크 A → B가 표현하는 것은?

A
A가 B의 부모 디렉토리임
B
A와 B 사이에 관계가 있음 (종류는 주변 텍스트로 전달)
C
A가 B에 의존함을 공식 선언
D
B의 데이터가 A에 포함됨
✅ 맞습니다! OKF 명세 §5.3: "링크는 관계를 주장(asserts a relationship)한다. 특정 관계 종류는 링크 자체가 아니라 주변 텍스트로 전달된다."
❌ OKF 링크는 타입이 없는 관계 엣지입니다. 의미는 마크다운 본문에서 읽어야 합니다.

Q2. ingredients/retinol.md에서 같은 디렉토리의 niacinamide.md로 절대 링크를 쓴다면?

A
[나이아신아마이드](niacinamide.md)
B
[나이아신아마이드](./niacinamide.md)
C
[나이아신아마이드](/ingredients/niacinamide.md)
D
[나이아신아마이드](ingredients/niacinamide.md)
✅ 맞습니다! 절대 링크는 항상 /로 시작하고 Bundle 루트 기준으로 전체 경로를 씁니다. 같은 디렉토리라도 동일합니다.
❌ 절대 링크는 항상 /로 시작합니다. Bundle 루트부터의 전체 경로를 적어야 합니다.
