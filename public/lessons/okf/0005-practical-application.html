실무 적용

업무 지식 관리 시스템을 OKF로 구축하는 로드맵

🗺️ 내 업무에 OKF 적용하기

목표: 화장품 기획 업무의 지식(성분, 제품, 프로세스, 캠페인)을 AI 에이전트와 팀원 모두가 접근할 수 있는 OKF Bundle로 구조화하기.

🧪

성분 지식베이스

  • INCI 명, 효능, EWG 점수
  • 배합 호환성 관계
  • 규제 정보 링크
🏷️

제품 카탈로그

  • SKU, 브랜드, 타겟
  • 핵심 성분 크로스링크
  • 출시 이력
📋

프로세스 플레이북

  • 안정성 테스트 절차
  • 승인 워크플로우
  • 규제 체크리스트
📣

캠페인 지식

  • 타겟 페르소나
  • 클레임 근거
  • 채널별 전략

🚀 5단계 구축 로드맵

1️⃣

Bundle 디렉토리 설계

어떤 개념 타입이 필요한지 결정. ingredients/, products/, processes/, references/ 등 서브디렉토리 구조 초안 작성.

⏱ 1~2시간
2️⃣

type 네이밍 컨벤션 정의

예: Cosmetic Ingredient, Product SKU, Process Playbook, Campaign Brief. 팀 전체에서 일관되게 사용할 type 값 목록 확정.

⏱ 30분
3️⃣

핵심 Concept 10개 작성

가장 자주 참조되는 성분 5개 + 핵심 프로세스 5개부터 시작. frontmatter + body 섹션(Schema, Examples, Citations) 포함.

⏱ 하루
4️⃣

Cross-link 연결 + index.md 생성

성분→제품, 제품→프로세스 링크 추가. 각 디렉토리에 index.md 생성. Git 레포지토리로 초기화 및 첫 커밋.

⏱ 반나절
5️⃣

AI 에이전트 연동 + 팀 공유

Claude에게 Bundle을 컨텍스트로 제공. Obsidian에서 마크다운으로 열기. 팀원에게 기여 방법 공유. log.md로 변경 이력 관리 시작.

⏱ 지속적

📁 화장품 브랜드 Bundle 예시

evas-cosmetic-knowledge/ ├── index.md # 전체 Bundle 목차 ├── log.md # 변경 이력 │ ├── ingredients/ │ ├── index.md │ ├── niacinamide.md # type: Cosmetic Ingredient │ ├── retinol.md │ └── ceramide-np.md │ ├── products/ │ ├── index.md │ ├── ceraclean-foam.md # type: Product SKU │ └── fraijour-cream.md │ ├── processes/ │ ├── index.md │ ├── stability-test.md # type: Process Playbook │ └── regulatory-approval.md │ ├── campaigns/ │ ├── index.md │ └── sensitive-skin-2026.md # type: Campaign Brief │ └── references/ ├── cosing-database.md # type: Reference └── kfda-guidelines.md
💡 AI 에이전트 활용

이 Bundle을 Claude에게 제공하면: "세라마이드 NP가 포함된 제품은?", "레티놀 배합 주의사항은?", "안정성 테스트 절차를 알려줘" 같은 질문에 정확하게 답변할 수 있습니다.

🔌 기존 도구와 통합

OKF는 기존 도구와 자연스럽게 통합됩니다:

  • Obsidian — OKF Bundle 디렉토리를 Vault로 열면 바로 브라우징 가능. [[링크]] 스타일도 함께 사용 가능.
  • GitHub/GitLab — 레포지토리에 Bundle을 체크인하면 PR 리뷰, 히스토리, blame이 자동으로 작동.
  • Claude/ChatGPT — Concept 파일들을 컨텍스트로 제공하면 지식 기반 Q&A 에이전트 즉시 구축.
  • MkDocs / Hugo — Bundle을 정적 사이트로 렌더링해 팀 위키로 제공 가능.
