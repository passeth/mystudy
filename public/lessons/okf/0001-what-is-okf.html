OKF란 무엇인가?

Open Knowledge Format — 지식을 마크다운으로 표현하는 보편적 형식

🤔 문제 인식: 왜 OKF가 필요한가?

조직에는 수많은 지식이 있습니다. 제품 스펙, 프로세스, 성분 데이터, 캠페인 노하우... 그런데 이 지식들은 흩어져 있습니다:

❌ 현재 문제
  • Notion에 일부, Slack에 일부
  • 사람마다 다른 형식
  • AI가 읽기 어려움
  • 버전 관리 불가
  • 이식 불가 (락인)
✅ OKF로 해결
  • 모두 마크다운 파일
  • 통일된 구조
  • AI가 직접 읽음
  • Git으로 버전 관리
  • 어디서든 열림

📖 OKF의 정의

OKF (Open Knowledge Format)는 지식을 YAML frontmatter가 있는 마크다운 파일로 표현하는 보편적이고 벤더 중립적인 형식입니다.

핵심 아이디어는 단순합니다:

  • 누구나 만들 수 있다 — 사람이 직접 쓰거나, AI 에이전트가 자동 생성
  • 누구나 소비할 수 있다 — 정적 파일 서버, Obsidian, LLM, 검색 인덱스
  • 특정 도구에 종속되지 않는다 — 그냥 텍스트 파일
💡 비유

OKF는 지식 세계의 HTML 같은 존재입니다. HTML이 "웹 페이지를 어떻게 표현할지"의 표준이듯, OKF는 "지식을 어떻게 파일로 표현할지"의 표준입니다. 특정 브라우저 없이도 HTML을 읽을 수 있듯, 특정 앱 없이도 OKF를 읽을 수 있습니다.

✨ OKF의 핵심 속성 8가지

👁️
Human-readable
SDK 없이 읽기 가능
🤖
Agent-readable
LLM이 직접 소비
📊
Version-controllable

🚀
Portable
디렉토리 하나가 전부
🔗
Graph-shaped
개념 간 링크로 그래프
🔧
Extensible
임의 필드 추가 가능

🗂️ OKF가 실제로 어떻게 생겼나?

아주 간단한 예시입니다. 성분 개념 문서를 OKF로 표현하면:

---
type: Cosmetic Ingredient
title: 나이아신아마이드 (Niacinamide)
description: 피부 미백 및 모공 축소에 효과적인 비타민 B3 유도체
tags: [미백, 비타민B3, 수용성]
timestamp: 2026-06-22T00:00:00Z
---

비타민 B3의 활성형으로, 멜라닌 이동을 억제하여 미백 효과를 낸다.

# 효능
- 멜라닌 이동 억제 → 미백
- 피지 분비 조절 → 모공 축소
- 표피 단백질 합성 촉진 → 장벽 강화

# 배합 주의사항
[비타민C](/ingredients/vitamin-c.md)와 함께 배합 시 반응 가능성 검토 필요.

# Citations
[1] [CosIng Entry](https://ec.europa.eu/growth/tools-databases/cosing/)

위 파일이 ingredients/niacinamide.md로 저장되면 그것이 하나의 Concept 문서입니다.

🧠 이해 확인 퀴즈

Q1. OKF에서 하나의 "지식 단위"를 무엇이라고 부르나요?

A
Bundle
B
Concept
C
Frontmatter
D
Tag
✅ 정답! Concept은 하나의 마크다운 파일로 표현되는 단일 지식 단위입니다. Bundle은 여러 Concept의 컬렉션이에요.
❌ 다시 생각해보세요. OKF 명세의 Terminology 섹션을 떠올려보세요. "A single unit of knowledge"는 무엇이라고 했나요?

Q2. OKF가 Git 버전 관리에 적합한 가장 핵심적인 이유는?

A
파일이 암호화되어 있어서
B
전용 Git 플러그인이 있어서
C
일반 텍스트 파일이라 diff/blame/PR이 그냥 작동해서
D
자동 커밋 기능이 내장되어 있어서
✅ 맞습니다! 바이너리 파일(Word, Excel)은 Git에서 diff가 불가능하지만, 마크다운 텍스트 파일은 한 줄 단위로 변경 내용을 추적할 수 있습니다.
❌ OKF는 전용 도구가 전혀 없습니다. 왜 "그냥 텍스트 파일"인 것이 강점인지 생각해보세요.

Q3. OKF 문서에서 반드시 있어야 하는 (REQUIRED) frontmatter 필드는?

A
title
B
description
C
type
D
tags
✅ 정확합니다! type만이 REQUIRED 필드입니다. title, description, tags, timestamp는 모두 권장(Recommended) 사항입니다.
❌ OKF 명세에서 Required와 Recommended의 구분이 중요합니다. type 하나만이 필수입니다!
