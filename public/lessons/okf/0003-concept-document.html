Concept 문서 작성법

YAML frontmatter + Markdown body — OKF의 핵심 단위 만들기

📄 Concept 문서의 두 파트

모든 Concept 문서는 딱 두 부분으로 구성됩니다:

--- ← frontmatter 시작 type: Cosmetic Ingredient # REQUIRED title: 나이아신아마이드 # 권장 description: 피부 미백 비타민 B3 # 권장 resource: https://cosing.eu/... # 권장 tags: [미백, 비타민B3] # 권장 timestamp: 2026-06-22T00:00:00Z # 권장 ewg_score: 1 # 임의 확장 --- ← frontmatter 끝 # 설명 비타민 B3의 활성형. 멜라닌 이동을 억제한다. # 효능 - 미백 효과 - 모공 축소 # Citations [1] [CosIng Database](https://cosing.eu)
🔴 REQUIRED
type — 유일한 필수 필드. 없으면 비준수(non-conformant).
🟡 RECOMMENDED
title, description, resource, tags, timestamp — 생략 가능하지만 강력 권장.
⚪ EXTENSION
임의 키 추가 가능. Consumers는 미알려진 키를 무시함.
🟢 BODY
자유로운 마크다운. 제목, 목록, 표, 코드블록 모두 가능.

🏷️ Frontmatter 필드 상세

필드필수 여부설명예시
type REQUIRED Concept의 종류. 중앙 등록 없이 자유롭게 정의. BigQuery Table, Metric, Playbook
title 권장 사람이 읽기 위한 표시 이름. 없으면 파일명에서 유도. Customer Orders
description 권장 한 문장 요약. index.md 자동생성과 검색에 사용. One row per completed order.
resource 권장 Concept이 설명하는 실제 자산의 URI. 추상 개념이면 생략. https://bigquery.../table
tags 선택 가로 분류용 태그 목록. [미백, 수용성]
timestamp 선택 마지막 수정 시각. ISO 8601. 2026-06-22T00:00:00Z

📝 Body 작성 관례

Body에 필수 섹션은 없습니다. 그러나 아래 헤딩은 관례적(conventional) 의미를 가집니다:

헤딩용도예시 맥락
# Schema자산의 컬럼/필드 구조적 설명테이블, API 응답 스키마
# Examples구체적 사용 예시 (코드블록 등)쿼리 예시, 배합 예시
# Citations본문 주장을 뒷받침하는 외부 출처논문, 공식 문서, 규제 데이터
💡 구조적 마크다운 권장

OKF 명세는 "구조적 마크다운(헤딩, 목록, 표, 코드블록)을 자유 형식 산문보다 선호하라"고 권장합니다. 이유: 구조가 있어야 사람과 LLM 모두 검색과 추출이 쉽습니다.

Q1. 다음 중 OKF v0.1 준수(conformant)를 위반하는 것은?

A
title 필드가 없는 문서
B
알 수 없는 type 값 (예: "My Custom Type")
C
frontmatter가 전혀 없는 .md 파일
D
임의의 추가 필드 (예: ewg_score: 1)
✅ 맞습니다! 모든 비예약 .md 파일은 파싱 가능한 YAML frontmatter 블록을 포함해야 합니다. frontmatter 자체가 없으면 비준수입니다.
❌ OKF §9 Conformance: title이 없거나, 미알려진 type이거나, 추가 필드가 있어도 모두 허용됩니다. 반드시 있어야 하는 것은 frontmatter 블록과 type 필드입니다.

Q2. resource 필드를 생략해도 되는 Concept의 예는?

A
BigQuery 테이블 문서
B
API 엔드포인트 문서
C
온콜 인시던트 대응 플레이북
D
S3 버킷 설명 문서
✅ 정확! OKF 명세: "resource 필드는 추상적 아이디어를 설명하는 Concept에서는 생략한다." 플레이북은 실제 물리 자산이 아닌 프로세스 지식이므로 생략 가능합니다.
❌ BigQuery 테이블, API 엔드포인트, S3 버킷은 모두 실제 자산(physical resource)입니다. resource 필드에 그 자산의 URI를 넣어야 합니다.
