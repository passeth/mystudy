📦 Bundle이란?

Knowledge Bundle은 자기 완결적인 지식 문서의 계층적 컬렉션입니다. 배포의 단위이자 지식의 "패키지"입니다.

Bundle은 단순히 마크다운 파일들이 있는 디렉토리입니다. 특별한 형식이 없습니다.

Bundle은 다음 방식으로 배포할 수 있습니다:

  • 🐙 Git 레포지토리 — 권장 방식. 히스토리, blame, PR 모두 작동
  • 📁 타볼/ZIP — 아카이브로 공유
  • 📂 큰 레포의 서브디렉토리 — 코드와 함께 관리

🗂️ Bundle 디렉토리 구조

아래는 화장품 지식 Bundle의 예시 구조입니다:

📌 예약 파일 (Reserved Filenames)

어느 디렉토리에서든 두 파일명은 특별한 의미를 가집니다. Concept 문서로 사용 금지.

파일명 용도 필수 여부
index.md 해당 디렉토리의 목차. Progressive disclosure를 위해 생성 — 전체 Bundle을 로드하지 않고도 탐색 가능하게 함. Optional
log.md 변경 이력. 날짜별로 어떤 Concept이 추가/수정/삭제되었는지 기록. Optional

index.md 형식

## 성분 (Ingredients) * [나이아신아마이드](niacinamide.md) - 피부 미백 및 모공 축소 비타민 B3 * [레티놀](retinol.md) - 주름 개선 비타민 A 유도체 ## 제품 (Products) * [세라클린 폼](../products/ceraclean-foam.md) - 저자극 클렌징 폼

log.md 형식

# Bundle Update Log ## 2026-06-22 * **Creation**: 나이아신아마이드 개념 문서 초안 작성 * **Update**: 레티놀 배합 주의사항 보완 ## 2026-06-15 * **Initialization**: Bundle 초기 구조 생성
💡 Progressive Disclosure

index.md의 핵심 가치: LLM이 전체 Bundle을 한번에 로드하지 않고도 "무엇이 있는지" 먼저 파악할 수 있습니다. 컨텍스트 창을 효율적으로 사용합니다.

Q1. ingredients/vitamin-c.md라는 파일이 있다면, 이 Concept의 ID는 무엇인가요?

A
vitamin-c
B
ingredients/vitamin-c.md
C
ingredients/vitamin-c
D
/ingredients/vitamin-c
✅ 맞습니다! Concept ID는 Bundle 루트로부터의 경로에서 .md 확장자를 제거한 것입니다.
❌ OKF 명세: "The path of the concept's file within the bundle, with the .md suffix removed." — .md를 빼고, 앞에 /는 없습니다.

Q2. index.md에서 frontmatter (YAML 블록)가 허용되는 위치는?

A
모든 index.md에서 허용
B
어떤 index.md에서도 허용되지 않음
C
Bundle 루트의 index.md에만 허용 (버전 선언용)
D
log.md에서만 허용
✅ 정확합니다! Bundle 루트 index.md에서만 okf_version: "0.1" 같은 frontmatter를 선언할 수 있습니다. 이게 Bundle의 버전을 선언하는 유일한 방법입니다.
❌ OKF 명세 §11 Versioning을 확인하세요. 특별히 허용되는 딱 한 곳이 있습니다.
