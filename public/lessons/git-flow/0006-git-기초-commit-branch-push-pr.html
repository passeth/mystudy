Lesson 0003

Git 기초: commit, branch, push, PR

Git은 “파일 변경 이력”을 관리하는 도구이고, GitHub는 그 이력을 팀이 같이 보고 검토하는 공간입니다. 처음에는 아래 6단계만 정확히 알면 됩니다.

비유로 이해하기

repository는 프로젝트 서랍장입니다. branch는 별도 작업 복사본입니다. commit은 저장된 스냅샷입니다. push는 내 스냅샷을 GitHub에 올리는 일입니다. PR은 그 변경을 공식 서랍장인 main에 넣어도 되는지 검토받는 요청입니다.

핵심 용어

용어쉬운 뜻언제 쓰나
repository프로젝트 전체 보관함GitHub의 passeth/riselab 같은 단위
main공식 버전 branch배포되거나 기준이 되는 안정된 줄기
branch작업용 줄기기능 추가, 버그 수정, 문서 수정
commit변경 저장점의미 있는 변경 묶음을 남길 때
pushGitHub에 올리기내 컴퓨터 commit을 원격 branch에 올릴 때
pullGitHub 변경 가져와 합치기다른 사람이 올린 변경을 내 컴퓨터에 반영할 때
PRmerge 검토 요청작업 branch를 main에 넣고 싶을 때
merge두 branch 합치기승인된 PR을 공식 branch에 반영할 때

초보자용 작업 흐름

1. 확인git status로 현재 변경을 봅니다.
2. branchgit switch -c codex/my-work
3. 수정파일을 바꿉니다.
4. commitgit add 후 git commit
5. pushgit push -u origin branch
6. PRGitHub에서 PR을 엽니다.
git status
git switch -c codex/my-small-change
git add path/to/file
git commit -m "Explain why this change exists"
git push -u origin codex/my-small-change

commit과 push의 차이

commit은 내 컴퓨터에 저장하는 일입니다. push는 그 저장점을 GitHub에 올리는 일입니다. commit만 하면 GitHub 화면에는 아직 보이지 않습니다.

push와 PR의 차이

push는 branch를 올리는 일입니다. PR은 올린 branch를 main에 합쳐도 되는지 검토받는 일입니다.

RISELAB에서 특히 중요한 규칙

지금 만든 운영 규칙은 main 직접 push를 피하고, 모든 변경을 PR로 검토하게 합니다. 그래서 작업자는 branch를 만들고, 테스트를 통과시키고, PR에서 approval을 받아야 합니다.

짧은 확인
