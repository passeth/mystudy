End-to-end MVP — Niacinamide 4% Face Cream으로 4-Tier 첫 관통

Tier 1·2·3·4를 각자 깊이 판 이후, 한 제품으로 처음부터 끝까지 연결한다. 목적은 "예쁜 시스템"이 아니라 가장 짧은 거리로 첫 SA 검토 sample을 산출하는 것.

이 레슨의 약속 마치면 (1) 8개 모듈로 분해된 MVP 아키텍처를 손에 쥐고, (2) 어떤 부분을 LLM·결정론·HITL로 가를지가 명확하며, (3) 첫 주에 만들 minimum surface와 두 번째 주에 추가할 surface를 분리할 수 있다. 가장 큰 함정은 "전부 한 번에"다. SA 검토 1건이 끝까지 나가는 가장 얇은 슬라이스가 목표.

1. MVP 범위 — "끝까지" 가는 가장 얇은 슬라이스

End-to-end MVP의 정의: 한 처방을 입력하면 한 PDF가 나오는 한 줄 파이프라인. Acceptance 측정·refiner 루프·HITL UI는 다 v2다.

MVP의 적 — perfectionism 앵커 제품 1개 + SCCS Opinion 5건만으로 시작. niacinamide는 SCCS Opinion이 없다 — 일부러 골랐다. 갭 처리(EFSA·CIR fallback)가 day-1부터 강제된다. Glycerin·Phenoxyethanol·Butylene Glycol·Tocopherol·Sodium Hyaluronate처럼 SCCS Opinion이 있거나 매우 친숙한 5개로만 처방을 짜라. 30개 짜리 실제 처방은 MVP 끝나고.

14개 성분 중 LLM 부담 Aqua / Sodium Hydroxide / Disodium EDTA는 toxicological assessment 통상 제외 ("considered safe under the conditions of use"). MVP에서 이걸 자동 skip하는 safe_list.csv를 두 시간 안에 만들면 LLM 호출이 14→11회로 즉시 30% 감소. Parfum은 IFRA 별도 처리 — MVP에서는 "Out of scope, IFRA Certificate of Compliance 첨부 전제" 한 줄로 처리.

4. Exposure Engine (M3) — 코드

from dataclasses import dataclass

@dataclass
class ExposureInput:
    pct: float          # 처방 % w/w
    a_g_per_day: float  # 일일 application amount (g/day)
    f_retention: float  # retention factor
    da_pct: float       # dermal absorption %
    bw_kg: float

def compute_sed(x: ExposureInput) -> float:
    """SED (mg/kg bw/day) per SCCS NoG 12th Rev §3-6.

    SED = (A * F * C * DA% / 100) / BW
    where C = pct/100 * 1000 (mg of substance per g of product)
    Combined: SED = A * F * (pct/100) * 1000 * (da_pct/100) / bw_kg
    """
    return (x.a_g_per_day * x.f_retention
            * (x.pct / 100.0) * 1000.0
            * (x.da_pct / 100.0)) / x.bw_kg

def compute_mos(noael_mg_per_kg_day: float, sed: float) -> float:
    """MoS = NOAEL / SED. SCCS NoG threshold = 100."""
    return noael_mg_per_kg_day / sed

이 코드의 모든 출력은 결정론적이다. LLM은 이 숫자를 만들지 않는다. narrative에 이 숫자가 들어갈 때 evaluator R1.2·R1.4가 literal match로 검증한다.

5. Tox Fetcher (M4) — RAG 최소 구성

MVP에서는 PDF 5건만 손으로 받아 메모리 벡터스토어에 적재. 호화 ASCII가 아닌 PDF text + page anchor가 핵심:

from pydantic import BaseModel, Field

class NoaelClaim(BaseModel):
    value: float = Field(description="NOAEL value in mg/kg bw/day")
    unit: str = Field(default="mg/kg bw/day")
    species: str
    duration_days: int | None
    study_design: str  # 'oral_28d', 'oral_90d', 'derm_28d', etc.
    source_id: str     # 'SCCS/1647/22' or 'EFSA Journal 2014;12(7):3759'
    page_anchor: str   # 'p.42 §4.3.2'

class DermalAbsorptionClaim(BaseModel):
    da_pct: float
    method: str        # 'OECD 428 in vitro', 'in vivo rat', etc.
    source_id: str
    page_anchor: str

class ToxFetchOutput(BaseModel):
    inci: str
    noaels: list[NoaelClaim]
    dermal: list[DermalAbsorptionClaim]
    confidence: Literal["high", "medium", "low"]
    gap_notes: list[str]

Claude API 호출 시 tool_use로 위 schema를 강제하고, system prompt에 박을 가드레일:

You extract toxicological claims from cosmetic safety documents.
ABSOLUTE RULES:
1. Only extract claims that are LITERALLY present in the provided text.
2. Every claim MUST include source_id and page_anchor. If you cannot
   identify a page or section, do NOT emit the claim.
3. Do NOT compute, infer, or convert units. Emit what is written.
4. If no NOAEL is found, return empty list and add gap_notes.
5. NEVER invent SCCS Opinion numbers.
Niacinamide MVP 노트 Niacinamide는 SCCS Opinion이 없다 → fallback 체인: EFSA Scientific Opinion (Vitamin B3 / niacinamide, EFSA Journal 2014;12(7):3759) → 경구 NOAEL 900 mg/kg bw/day → CIR (2005) 대조. 경구→피부 route extrapolation은 MVP에서 자동 안 한다. 그저 narrative에 "경구 NOAEL을 보수적으로 그대로 사용했다"를 명시. 진짜 PBPK conversion은 v3 이후.

6. Generator (M6) — 한 호출당 한 성분

MVP에서 절대 하지 말 것: "이 처방 전체 narrative를 한 번에 써줘". 한 호출 한 성분, 14개 호출. 이유: (a) diff·rerun이 쉽다 — 한 성분만 실패하면 그것만 재호출, (b) boilerplate 감지가 쉽다 — 호출이 독립이면 LLM이 의도적으로 다양화하지 않는 한 문체 중복이 자연스럽게 드러난다, (c) 비용·레이트리밋이 예측 가능하다.

def generate_substance_narrative(bundle: SubstanceFactBundle) -> str:
    response = anthropic.messages.create(
        model="claude-opus-4-7",
        max_tokens=1200,
        system=SCCS_INTERNAL_KOREAN_STYLE,  # Lesson 0006 Q4 mode
        messages=[{
            "role": "user",
            "content": build_user_prompt(bundle),
        }],
    )
    return response.content[0].text

def build_user_prompt(b: SubstanceFactBundle) -> str:
    return textwrap.dedent(f"""
    아래 fact bundle을 바탕으로 CPSR Part B.3 Reasoning 1개 성분 narrative를 작성하라.

    [성분 ID]
    INCI: {b.inci_canonical}
    CAS: {b.cas or '미확인'}

    [노출]
    농도: {b.concentration_pct}%
    SED: {b.sed_mg_per_kg_day} mg/kg bw/day
    가정: A={b.sed_assumptions.a}, F={b.sed_assumptions.f},
         DA%={b.sed_assumptions.da_pct}%, BW={b.sed_assumptions.bw_kg}kg

    [독성]
    NOAEL: {b.noael_chosen.value} {b.noael_chosen.unit}
    출처: {b.noael_chosen.source_id} ({b.noael_chosen.page_anchor})
    Confidence: {b.data_source_confidence}
    Gap notes: {b.data_gap_notes or '없음'}

    [위험]
    MoS: {b.mos}
    임계: {b.mos_threshold} (additional UF: {b.additional_uf_applied})

    제약: 본문에 위 수치와 출처를 모두 literal로 포함. 숫자를 변경 X.
    "[성분]은 안전하다" 식 단정 X. "조건 하에서 안전 여유가 확보되었다" 화법.
    """)

7. Evaluator (M7) — 코드 rubric만

Lesson 0006의 rubric 중 코드로 가능한 것만 MVP에 구현. LLM evaluator는 v2.

def evaluate_per_substance(narrative: str, b: SubstanceFactBundle) -> EvalReport:
    violations = []

    # R1.1 NOAEL source ID literal
    if b.noael_chosen.source_id not in narrative:
        violations.append(("R1.1", "NOAEL source_id missing"))

    # R1.2 NOAEL value (normalize comma + float compare)
    if not contains_number(narrative, b.noael_chosen.value, tol=0.001):
        violations.append(("R1.2", "NOAEL value not literal"))

    # R1.3 SED assumptions: at least 3 of A/F/DA/BW
    sed_a = b.sed_assumptions
    hits = sum([
        contains_number(narrative, sed_a.a, tol=0.01),
        contains_number(narrative, sed_a.f, tol=0.001),
        contains_number(narrative, sed_a.da_pct, tol=0.1),
        contains_number(narrative, sed_a.bw_kg, tol=0.1),
    ])
    if hits < 3:
        violations.append(("R1.3", f"only {hits}/4 SED inputs surfaced"))

    # R1.4 MoS value + threshold word
    if not contains_number(narrative, b.mos, tol=0.1):
        violations.append(("R1.4a", "MoS not literal"))
    if not any(w in narrative for w in ["임계", "threshold", "100"]):
        violations.append(("R1.4b", "threshold not mentioned"))

    # R1.6 gap honesty
    for note in b.data_gap_notes:
        keyword = extract_keyword(note)  # e.g. "SCCS Opinion 부재"
        if keyword not in narrative:
            violations.append(("R1.6", f"gap not surfaced: {note}"))

    score = max(0.0, 1.0 - 0.1 * len(violations))
    return EvalReport(score=score, violations=violations, narrative=narrative)

11. 위험과 회피

위험발현 형태회피
RAG가 page anchor 없이 NOAEL 토출narrative에 "according to SCCS Opinion"만 나옴, ID 없음schema에 page_anchor required. anchor 없으면 emit 자체 거부 (M4 단계).
LLM이 수치 변형 (900 → 900.0 → "approximately 900")R1.2 failsystem prompt에 "numbers verbatim". evaluator는 normalize 비교 (Lesson 0006 Q1).
14성분 호출 비용 폭주한 처방 $5+ → 100건이면 $500+/월safe_list로 즉시 -30%. v2에서 prompt caching·batch API 도입.
경구→피부 NOAEL 변환 무지"oral 900 mg/kg bw/day를 그대로 dermal MoS에 사용" 인식 부재 → SA 검토 시 즉시 거부narrative에 "경구 NOAEL을 보수적으로 그대로 사용" 명시. v2에서 oral-to-dermal correction (예: 100% bioavailability assumption) 옵션화.
처방 외부 전송 정책 미확인사내 처방을 외부 API로 보냈는데 사후 적발코드 작성 전 정책 확인. 미확인 시 Anthropic bedrock·vertex private deployment 또는 사내 LLM 사용.

12. v1 완료 정의 (Done의 정의)

  1. 처방 YAML 1개 → output/.../draft.docx 1개 생성 (manual gate 없이).
  2. 모든 성분의 SED·MoS가 deterministic + traceable (가정 표 docx에 동봉).
  3. narrative에 등장한 모든 수치는 fact bundle의 수치와 100% 일치 (M7 R1.2 fail = 0).
  4. MoS < 100인 성분이 있을 경우 generator 호출 없이 HITL 메시지 + 종료.
  5. 사내 SA 1명이 draft를 5분 읽고 "전체 형식·논조에 대한 first impression"을 글로 남김.

이 5개를 통과하면 v1 끝. acceptance rate는 v1에서 측정하지 않는다 — 측정은 reference set이 있어야 가능 (Lesson 0008).

13. Quiz — 4문항

Q1. 처방의 Aqua 70.5% + ... + Disodium EDTA 0.05% 14개 성분에 대해 M4 tox_fetcher를 14회 호출하려는 계획이다. 이게 왜 잘못된 자원 배분인가? 어떤 처리가 더 합리적인가?

답 보기
Aqua·Sodium Hydroxide·Disodium EDTA는 CPSR 관행상 "considered safe under conditions of use"로 별도 NOAEL/MoS 분석 없이 처리한다. tox 분석이 불필요한 성분은 safe_list.csv에 명시하고 M4 호출 자체를 건너뛴다. 즉시 14→11회로 줄어든다. Parfum은 IFRA Certificate of Compliance + 26 declared allergens 별도 라벨 의무 — MVP에서는 "out of scope, IFRA CoC 첨부 전제" 한 줄로 처리하고 자동 호출 X. 결과적으로 LLM 호출은 10건 이내. 모든 화장품 시스템은 "평가할 성분"과 "관행상 평가 면제 성분"의 구분이 day-1 자산이다 — 이 list는 사내 SA로부터 받는 첫 번째 ground truth.

Q2. Niacinamide는 SCCS Opinion이 없어 EFSA의 경구 NOAEL 900 mg/kg bw/day를 사용했다. 이 값을 dermal exposure 기반 MoS 계산에 그대로 적용하는 것이 보수적인가, 과소평가인가?

답 보기
대체로 보수적이다 — 하지만 모든 경우에 옳지는 않다. SCCS NoG는 dermal route 데이터가 없을 때 경구 NOAEL을 사용 가능하다고 명시하나, dermal absorption(DA%)을 통한 보정이 전제다. niacinamide의 경구 흡수율은 거의 100%(고bioavailability vitamin)이므로 경구 NOAEL을 systemic NOAEL로 간주할 수 있다. dermal 흡수율이 50% default라면 dermal exposure는 경구보다 노출이 낮으므로 같은 NOAEL 비교는 보수적이다. 그러나 경구 흡수율이 낮은 성분(예: 일부 polymer)은 경구 NOAEL이 systemic NOAEL을 과소평가 → dermal MoS를 부풀린다. MVP에서는 "경구→피부 그대로 사용" 보수적 가정으로 시작하되, narrative에 "extrapolation 미실시, 보수적 가정" 명시. v3에서 (a) systemic bioavailability factor (SCCS NoG §3-6.4) 또는 (b) PBPK modeling으로 route correction 도입.

Q3. Day 6에 SA에게 draft를 보내는 게 너무 이르다고 느낀다. "성분 14개 중 8개만 narrative 있고, A.2·A.3·A.8은 placeholder다. SA가 형편없다고 생각할 것 같다." 이 우려에 어떻게 대처할 것인가?

답 보기
대처하지 마라 — 그대로 보내라. 단, 이메일에 "이건 v1, 30% 완성도, 목적은 형식·논조 first impression. 빠진 섹션은 의도된 것, 채울 계획은 보유. 5~10분 정도만 읽고 '시스템이 잘못 향하고 있는 부분'만 짚어달라."를 명시. 완벽주의는 솔로 메이커의 핵심 적이다. SA가 "narrative 화법이 사내 표현과 다르다"를 day 7에 짚어주면 v2 전체 generator system prompt가 정확한 방향으로 간다. 90%로 끌어올린 뒤 받는 같은 피드백은 한 달 늦다. 더 작은 우려: SA가 화났다? SA의 시간을 가치 있게 쓰는 책임은 당신에게 있다 — 5분 검토 + "이 방향이 맞다/틀리다" 코멘트만 요청하라. 30분 짜리 라인 by 라인 review는 v2 이후.

Q4. M6 generator를 호출할 때 사내 처방 데이터(자사 INCI %)를 Anthropic API로 그대로 전송할 것인가? 어떤 옵션이 있는가?

답 보기
법무·정보보안 정책 확인이 코드 작성보다 먼저다. 옵션: (1) OpenAI/Anthropic public API + 0-day retention + 학습 opt-out — 가장 간단, 정책이 허용하면 v1에 적합. 단 처방 % 자체는 영업비밀이라 거부될 가능성 높음. (2) AWS Bedrock의 Anthropic Claude — VPC 내 호출, 회사 AWS 계정에 격리, 가장 흔한 enterprise 정답. (3) Google Vertex AI의 Anthropic Claude — 회사 GCP 사용 시. (4) Azure OpenAI — Anthropic 대신 GPT-4 사용해야 함. (5) 처방 abstraction — INCI 이름과 농도를 generator에 보내고 "실제 제품명·브랜드"는 마스킹. 가장 보수적. (6) 온프레미스 LLM — Llama·Qwen 등 self-hosted. CPSR 작성 품질에서는 v3 이후 검토. MVP는 (2) Bedrock + (5) abstraction 조합이 권장. 결정은 RA·법무·정보보안 합의 후 NOTES.md에 기록.
