SUNSCREENLesson 0010 · 약 40분

Sunscreen·UV Filter 확장 — Nano + Photo Toxicity의 추가 복잡성

Niacinamide cream까지의 모든 도메인은 sunscreen 앞에서 다시 짜야 한다. UV filter는 (1) Positive list 강제, (2) Nano material 별도 의무, (3) Photo toxicity·photostability 평가, (4) SPF·UVA-PF 효능 표시 검증. 4개 축 각각이 별도 sub-pipeline.

이 레슨의 약속 마치면 (1) UV filter의 Annex VI Positive list 강제가 시스템 architecture에 무엇을 추가하는지, (2) Nano TiO₂·Nano ZnO에 EU Reg Art.16 별도 의무가 어떻게 작용하는지, (3) Photo toxicity(MPE·PIF·UVA-induced)·photostability 평가가 fact bundle에 추가되는 fields, (4) SPF·UVA-PF·broad-spectrum claim 검증의 시스템 위치. Leave-on 스킨케어와 같은 prompt·rubric으로 sunscreen을 처리하면 즉시 SA가 거부한다.

1. 무엇이 다른가 — 4축 추가 의무

축Leave-on 스킨케어Sunscreen 추가
1. 성분 listAnnex III (제한 사용 성분) 체크Annex VI (UV filter Positive list) 강제 — list 밖 UV filter는 사용 불가
2. Nano 의무Annex III·VI에 nano 별도 entryEU Reg Art.16 통보 + nano-specific SCCS Opinion + CPNP notification 6개월 전
3. 노출·MoSSED·MoS·threshold 100+ photo-induced toxicity (MPE - Minimal Phototoxic Effect, PIF - Photo-Irritation Factor) + photostability 평가
4. 효능 표시실증 없는 한 claim XSPF·UVA-PF·broad-spectrum·water-resistant 각각 ISO test로 실증 의무
Sunscreen 진입 = 별도 시스템 모듈 4개 추가 Niacinamide cream 시스템을 그대로 sunscreen에 돌리면 SA가 4축 모두에서 갭을 지적한다. 새 모듈: M9 Annex VI Positive Check / M10 Nano Form Detection / M11 Photo Tox Engine / M12 SPF Claim Validator. 각각 결정론·LLM·HITL 분할이 다시 필요하다. 이 레슨은 sunscreen MVP를 새 미션으로 다룬다 — niacinamide MVP의 확장이 아니라 별도 phase.

2.1 M9 Annex VI Positive Check — 시스템 모듈

def check_annex_vi_compliance(formula: list[FormulaRow]) -> AnnexViReport:
    """처방의 UV filter가 Annex VI Positive list에 있는가 검증."""
    violations = []
    uv_filters_in_formula = [
        r for r in formula
        if r.cosing_functions and "uv_filter" in r.cosing_functions
    ]

    for uv in uv_filters_in_formula:
        annex_vi = cosing.lookup_annex_vi(uv.inci, uv.nano_form)

        if annex_vi is None:
            violations.append((
                "NOT_LISTED",
                f"{uv.inci}{'  (nano)' if uv.nano_form else ''} not in Annex VI Positive list",
            ))
            continue

        if uv.pct > annex_vi.max_pct:
            violations.append((
                "OVER_LIMIT",
                f"{uv.inci} at {uv.pct}% exceeds Annex VI {annex_vi.max_pct}%",
            ))

        for cond in annex_vi.conditions:
            if not satisfies(formula, cond):
                violations.append(("CONDITION", f"{uv.inci}: {cond.description}"))

    return AnnexViReport(violations=violations, hitl_required=bool(violations))
"UV filter가 아닌 성분이 UV blocking 효과를 낸다"의 함정 Iron oxide는 UV filter로 Annex VI에 없지만 자외선 차단 효과가 있어 tinted sunscreen에 자주 사용. 이 경우 (a) 유효 성분으로 표시 X, (b) iron oxide는 colorant Annex IV로 별도 처리. 시스템에서 "UV filter 후보지만 Annex VI에 없는 성분"이 발견되면 generator에게 "이 성분을 SPF 기여 항으로 묘사 X" 가드 주입. 잘못된 묘사는 false claim → 규제 risk.

3. Nano Material — EU Reg Art.16 별도 의무

EU Reg 1223/2009 Art.16: nano material 함유 화장품은 출시 6개월 전 CPNP nano notification. CPSR에는 별도 nano section.

3.1 Nano의 정의 (Reg 1223/2009 Art.2)

"이용자 친화적으로 변형된 정의" — 다음 3 조건 모두 충족 시 nano:

  • 의도적으로 제조됨 (insoluble or biopersistent)
  • 1~100 nm 크기의 입자를 포함
  • 외부 또는 내부 표면적 비율이 일정 기준 초과 (구체 수치는 EC Recommendation 2011/696/EU)

3.2 시스템에 필요한 추가 fact

class NanoMaterialFact(BaseModel):
    inci: str
    is_nano: bool
    primary_particle_size_nm: Optional[tuple[float, float]]  # (D50, D90)
    crystalline_form: Optional[str]  # "rutile", "anatase"
    surface_coating: list[str]            # SCCS가 검증한 coating 만 허용
    coating_pct_w_w: Optional[float]
    purity_pct: Optional[float]
    aggregate_state_data: Optional[dict]  # aggregation 정도
    biopersistence_data: Optional[dict]
    sccs_opinion_id: str                       # SCCS opinion for this specific nano form
    coa_ref: str                               # 사내 incoming COA

# Nano formula row는 일반 FormulaRow에 nano fact 첨부
class FormulaRow(BaseModel):
    inci: str
    pct: float
    nano: Optional[NanoMaterialFact] = None  # None이면 non-nano

3.4 Nano 정당화 narrative — 추가 섹션

Nano 함유 CPSR은 B.3에 nano-specific subsection이 들어간다:

B.3.X Nano Material Safety Assessment — Titanium Dioxide (nano)

본 제형에 사용된 Titanium Dioxide (nano)는 다음 조건을 충족한다:

1. SCCS Opinion 부합: SCCS/1583/17 (TiO₂ nano in sunscreen) §5 결론에 따라
   본 입자 사양은 안전한 사용 범위 (rutile crystalline, ≥99% purity,
   alumina/silica coating, D50 30-50nm)를 충족한다.

2. Annex VI Entry X3a 부합: 25% 한도, coating 종류 명시 (alumina, silica,
   stearic acid 등 SCCS 인정 list 내).

3. CPNP Notification: 2025-12-15 EU notification 완료 (notification ID:
   CPNP-2026-XXX), 출시 6개월 전 의무 이행.

4. Aggregate vs Primary: 본 제형 내 dispersion 안정성 시험 (사내 R&D
   2025-INT-052) 결과, aggregate 형태가 우세하여 primary nano particle
   노출은 제한적임.

5. Inhalation 위험 회피: 본 제품은 cream 제형 (spray 아님). SCCS 견해에
   따라 inhalation route 비고려.

광 안전성: SCCS Opinion에 따라 본 nano TiO₂ coating은 photoreactive
species 생성을 억제한다. 사내 photostability 시험에서 4시간 일광 노출 후
SPF 잔존율 ≥ 90% 확인 (사내 R&D 2025-INT-053).

이 narrative의 모든 사실은 fact bundle의 NanoMaterialFact + 사내 R&D Report에서 온다. LLM은 묶기만 한다.

4. Photo Toxicity — Tier 3 확장

일반 toxicity 위에 UV·HEV 광 노출 시 추가 위험 평가. UV filter 자체가 광흡수 분자이기 때문.

4.1 핵심 평가 항목

항목설명test method
Photo-irritation (PIF)UV 노출 시 자극 강화 여부OECD TG 432 (3T3 NRU phototoxicity test, in vitro)

4.3 Octocrylene 사례 — Benzophenone 분해

2021~2023 EU에서 Octocrylene이 시간·UV 노출 시 Benzophenone으로 분해 → Benzophenone이 발암 우려 (NTP 2021 분류). SCCS는 그러나 SCCS/1647/22에서 검토 후 일정 농도 이하 OK로 결론. 시스템 시사점:

  • UV filter의 분해산물 list를 fact bundle에 의무 포함.
  • 분해산물 각각의 NOAEL·발암성 평가가 fact_completeness에 들어감.
  • SCCS Opinion의 최신 결론(분해산물 포함 결론)을 narrative에 명시.
  • SCCS Opinion이 1년 이내에 갱신된 UV filter는 "최신 opinion review status" 라벨 — generator가 무시하지 못하게.
시스템은 SCCS Opinion 갱신을 어떻게 알 수 있는가 SCCS Opinion 페이지에 RSS 또는 publication date 메타. 주 1회 cron으로 corpus rebuild. UV filter 관련 opinion이 갱신되면 해당 성분을 사용한 모든 SKU에 "dirty" 라벨 → SA 큐에 알림. Lesson 0006 Section 6의 trigger 표 + Worldover 같은 enterprise tool의 "regulation change alert"를 우리 시스템 안에 압축.

5. SPF·UVA-PF Claim Validation — M12

Sunscreen은 효능 claim이 강제 규제 대상. CPSR Part A.5·B.1에 efficacy data 인용 필수.

Claim실증 방법표시 의무

UVA-PF (UVA Protection Factor)ISO 24443 in vitroUVA-PF ≥ SPF/3 (EU 권고)

def validate_spf_claims(claims: ProductClaims, efficacy_data: EfficacyData) -> ClaimReport:
    violations = []
    if claims.spf is not None:
        if not efficacy_data.iso_24444_report:
            violations.append(("NO_ISO_24444", "SPF claimed without in vivo data"))
        elif abs(claims.spf - efficacy_data.iso_24444_report.measured_spf) > 5:
            violations.append(("SPF_MISMATCH", "Claimed SPF vs measured deviation"))

    if claims.broad_spectrum:
        ratio = efficacy_data.uva_pf / efficacy_data.spf
        if ratio < 1/3:
            violations.append(("UVA_RATIO", f"UVA-PF/SPF = {ratio:.2f} < 1/3"))

    if claims.water_resistant and not efficacy_data.water_resistant_minutes:
        violations.append(("NO_WATER_RESIST_DATA", ...))

    if "reef safe" in claims.free_text_claims.lower():
        violations.append(("REEF_SAFE_UNDEFINED", "reef-safe lacks legal definition"))

    return ClaimReport(violations=violations, hitl_required=bool(violations))

6.1 Sunscreen Application Amount (A)

Leave-on face cream A=1.54 g/day 대신 sunscreen은 전신 적용:

제품A (g/day)FSCCS NoG ref
Face cream (leave-on)1.541.0§3-6.4 Table 4
Body lotion7.821.0§3-6.4 Table 4
Sunscreen (전신)18.0 (성인 7.5 mg/cm² × 24000 cm²/1000)1.0§3-9 sunscreen-specific
Lip balm0.0571.0§3-6.4

SED 식: SED = A × F × (pct/100) × 1000 × (DA%/100) / BW. Sunscreen은 A가 face cream의 ~12배 → 같은 농도여도 SED 12배 → MoS 1/12. 임계 100을 통과하려면 NOAEL이 12배 더 높거나 농도가 12배 낮아야 한다. 이게 UV filter가 NOAEL 수천 mg/kg bw/day인 성분만 commercially viable한 이유.

9. CPNP Nano Notification 자동화 — out of scope (in v1)

CPNP nano notification 자체는 EC 공식 portal에 6개월 전 사람이 직접 입력. 시스템이 자동 제출하지 X (현재 API 없음). 시스템이 도울 수 있는 것:

  • Notification 입력에 필요한 모든 fact 정리 (primary size, coating, purity, COA, SCCS opinion ref).
  • Notification deadline 추적 ("출시 예정일 - 6개월" alert).
  • 제출 완료 후 notification ID 저장 → 모든 CPSR에 자동 inject.
  • Notification 제출 정보를 사내 dashboard에 일원화.

12. Quiz — 5문항

Q1. Niacinamide cream의 SED·MoS 임계 100 통과 패턴이 sunscreen에 그대로 적용 가능한가? Mineral TiO₂ (nano) 12%가 sunscreen에 들어있을 때 단순 MoS 계산이 의미 있는가?

답 보기
부분적으로만 의미 있다. Nano TiO₂는 dermal absorption이 극히 낮다 (SCCS/1583/17: intact skin 통과 본질적으로 X — < 0.0001%). 즉 default DA% 50% 적용은 과보수적이며 SED를 ~5000배 부풀린다. (a) SCCS-specific DA% 사용: nano TiO₂는 0.01% 이하로 SCCS opinion이 명시. fact bundle의 da_pct는 SCCS opinion 값 사용 (default 아님). (b) 그래도 SED·MoS 계산 자체는 수행: 형식적 무결성 + audit trail. (c) narrative에 "DA% 0.01% (SCCS/1583/17 §4)" 명시: SA가 default 적용을 안 한 이유를 보여줌. (d) 진짜 평가는 inhalation (spray 제형이면 P0) + photoreactivity (coating 검증) + aggregate exposure. Nano sunscreen의 risk는 systemic toxicity가 아니라 photoreactive ROS·inhalation route.

Q2. 사내 R&D가 새 처방의 COA에 "TiO₂ D50 = 110nm"으로 기재했다. 정의상 1~100nm 입자 포함 시 nano인데 D50이 110nm이면 입자의 절반 이상이 100nm 초과다. 그래도 nano로 처리해야 하는가?

답 보기
D50만으로 결정 불가 — D10 또는 분포 확인 필수. EU 정의는 "1~100nm 입자를 포함" — 개수 분포 기준. D50=110nm여도 D10이 60nm면 입자의 10% 이상이 100nm 이하 → nano로 분류. EC Recommendation 2011/696/EU는 "입자수의 50% 이상이 1~100nm" 기준을 제시하지만 화장품 Reg 1223/2009 정의는 더 광범. 시스템 처리: (a) COA에 D10·D50·D90 모두 의무. D10 데이터 없음 → P0 HITL. (b) R&D에 입자 size distribution 전체 요청. (c) 의심스러우면 nano로 처리: CPNP notification 추가 작업이 시장 회수보다 압도적으로 저렴. (d) SA + R&D 합의 후 결정을 audit log에 기록. "D50만 봤다"는 audit 시 결함.

Q3. 시스템이 SCCS Opinion update를 주 1회 확인한다. 한 sunscreen SKU에 사용된 UV filter A의 SCCS Opinion이 갱신됐고 새 결론은 "기존 농도 한도 5% → 3%로 인하" 권고다. 현 처방은 4%다. 시스템은 무엇을 해야 하는가?

답 보기
5단계 자동 처리. (1) SKU에 "regulatory_dirty" 라벨: 새 SCCS Opinion 적용 시점부터 기존 CPSR이 outdated. (2) SA P0 알림: 슬랙 + 이메일. "UV filter A의 SCCS Opinion 갱신. 권고 한도 3%, 현 처방 4%. CPSR 재평가 필요." (3) 처방 검토 단계 자동 시작 X: 5%에서 3%로의 변경은 R&D·marketing·supply chain에 영향. SA가 "재평가 진행" 결정한 후에만 자동 재계산. (4) 대안 정보 제공: 시스템이 "다른 SKU 5건이 같은 UV filter 사용" + "4%에서 3%로 인하 시 SPF 영향 추정 (ISO 24443 in vitro 사전 검토 가능)" 표 제공. (5) EU 시행일 추적: SCCS Opinion이 EU Reg 개정으로 법제화되는 시점 (보통 12~24개월). 그 전까지 현 처방 합법, 그러나 새 처방·revision은 이미 신 기준 따라야. SA에게 timeline 명시. 시스템 가치 = SA가 SCCS 변경을 놓치지 않게 하는 것; ToxTool 같은 외부 도구는 "사내 SKU 영향 분석"이 약함.

Q4. Sunscreen narrative에 "Reef-safe, marine-friendly formulation"이라는 marketing claim이 들어왔다. R3.8 룰은 이를 감점 처리한다. 시스템은 generator에게 어떻게 가드를 박는가? 결정은 누가 하는가?

답 보기
3단계 가드. (1) Claim Validator (M12): free-text claims에서 "reef-safe" 또는 동등 표현 detection. legally undefined 용어 list에 매칭 (현재 hawaii·일부 카리브 지역 외엔 법적 정의 없음). (2) Generator system prompt에 명시 가드: "다음 claim들은 EU/한국/TFDA 법적 정의가 없으므로 narrative·결론에 사용 금지: 'reef-safe', 'marine-friendly', 'ocean-safe', 'biodegradable' (조건부)." Generator가 marketing brief를 받더라도 narrative에 옮기지 않음. (3) HITL trigger: claim list에 정의 없는 표현이 marketing에서 왔다면 SA + marketing 합의 큐. SA가 "marketing이 자기 책임으로 외부 표기, CPSR에는 제외" 결정 시 narrative에서 제외. 또는 "지역별 검토" 결정 시 hawaii용 별도 SKU 분리. 결정은 SA가 marketing과 합의 후 audit log에 기록. 시스템이 marketing의 자기 결정을 거부하지 X — 단 CPSR(법적 문서)에 들어가지 않음을 보장. marketing claim ≠ regulatory claim의 분리가 명확.

Q5. Niacinamide leave-on MVP의 generator system prompt를 그대로 sunscreen에 사용하면 어떤 실패가 나는가? 적어도 5개 나열.

답 보기
5+ 실패. (1) A=1.54 g/day 가정 적용: sunscreen은 18 g/day. SED 12배 과소평가 → MoS 12배 과대평가 → 실제 위험한 농도가 안전으로 분류. (2) Nano subsection 생성 X: 일반 leave-on prompt는 nano를 모름. R3.3 fail. (3) Annex VI 인용 안 함: 일반 leave-on은 Annex III·V·VI를 모두 같이 처리. UV filter의 positive list 의무가 본문에 안 들어감. R3.1 fail. (4) Photostability·degradation 진술 X: 일반 toxicity narrative만 생성. R3.5·R3.6 fail. (5) SPF claim validation 누락: claim 진술이 있어도 ISO 24444 인용이 없음. R3.7 fail. (6) "reef-safe" 같은 marketing claim 본문 침투: 가드가 없으므로 LLM이 마케팅 표현 자유 사용. R3.8 fail. (7) Application amount 본문 미노출: face cream A=1.54는 자명해 생략 가능하나 sunscreen A=18 g/day는 본문 의무. R3.2 fail. 해결: sunscreen 전용 system prompt + R3 rubric. 같은 generator 모델이라도 prompt만 갈아끼우면 충분 — 다른 LLM 필요 X.

13. 다음 단계 (이 미션의 범위 너머)

  • Chemical UV filter 확장: Octocrylene·Avobenzone·Ensulizole 등. 분해산물 추적 + photostability 시험 데이터 누적이 전제.
  • Tinted sunscreen: Iron oxide·Mica 등 colorant Annex IV 평가 분기. UV blocking 효과 misrepresentation 가드.
  • Spray sunscreen: inhalation route 별도 평가. SCCS Opinion 별도 검토.
  • Korean MFDS 기능성 화장품 자외선차단: EU와 별도 기능성 심사 process. KFDA-specific 화법 anchor 필요. (현 미션 out of scope, NOTES.md 항목)
  • Taiwan TFDA sunscreen: positive list 차이, nano 등록 의무 별도. TFDA PIF 중문 가이드라인 정독 (NOTES.md 항목).
