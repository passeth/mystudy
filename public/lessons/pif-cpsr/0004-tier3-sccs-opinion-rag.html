TIER 3Lesson 0004 · 약 35분

SCCS Opinion 코퍼스의 해부

NOAEL과 SCCS 결론을 1차 자료에서 fabrication 0%로 추출하는 RAG 설계. Tier 4 reasoning이 신뢰할 수 있는 ground truth를 만든다.

이 레슨의 약속 마치면 (1) SCCS Opinion 한 편의 표준 섹션 구조를 외워서 NOAEL이 어디에 있는지 5초 내 짚고, (2) Pydantic 스키마와 prompt 전략을 머릿속에 갖추며, (3) 일반 RAG가 화장품 평가에서 실패하는 4가지 모드와 그 방어 패턴을 안다. ToxTool이 in silico로 우회하는 그 자리에 우리는 grounded extraction을 박는다.

1. 왜 일반 RAG로는 안 되는가

일반 RAG(chunk → embed → top-k retrieve → LLM answer)는 화장품 안전성 자료에서 다음 4가지 실패가 나온다:

실패 모드왜 일어나나SA가 보는 결과
Fabricated NOAELLLM이 chunk 사이의 빈자리를 "그럴듯한" 숫자로 채움. RAG context에 없는 NOAEL을 만들어냄.인용 따라 들어갔는데 원문에 없는 숫자 → 신뢰 0.
잘못된 종 적용rat 시험 NOAEL인데 종 정보 없이 출력 → "human NOAEL"로 해석.MoS 계산 자체가 무효.
단위 혼동mg/kg bw/day vs μg/kg bw/day. PDF 본문에 단위가 "그림 캡션"이나 "주석"에 있고 본 문장에는 숫자만.1000배 차이 → MoS 100 vs 0.1.
보수 값 선택 실패한 성분에 여러 시험·여러 NOAEL이 보고됨. 가장 보수적(=낮은) 값을 골라야 하는데 LLM이 평균 또는 가장 그럴듯한 값을 고름.SA 검토 시 "왜 이 NOAEL을 골랐나" 질문 → 답 못함 → 거부.
설계 원칙 Tier 3는 사실 추출(extraction)만 한다. 해석·종합·결론은 Tier 4. 추출의 모든 사실에는 출처 anchor(문서 ID + 페이지/문단 + literal quote)가 강제. quote가 원문에 literal match 안 하면 evaluator가 거부.

2. SCCS Opinion의 표준 해부

SCCS Opinion은 1970년대부터 ~50년간 일관된 템플릿을 유지해왔다. 모든 Opinion이 거의 동일한 섹션 번호를 가지며, 이것이 RAG schema 설계의 가장 큰 자산이다.

전형적 Opinion 구조 (요약)

1. Background 집행위원회 mandate 배경
2. Terms of reference 집행위가 SCCS에 묻는 정확한 질문
3. Opinion 평가 본체 (이하 모든 sub-section)
3.1 Chemical and Physical Specifications 물질 정체
3.1.1 Chemical identity CAS, EC, INCI, structural formula — Tier 2 lookup 필수 매칭
3.2 Function and uses
3.3 Toxicological Evaluation ⭐ Tier 3 추출의 메인 사냥터
3.3.1 Acute toxicity
3.3.2 Irritation and corrosivity
3.3.3 Skin sensitisation
3.3.4 Dermal / percutaneous absorption ⭐ DAp 추출 → Tier 1로 feed
3.3.5 Repeated dose toxicity ⭐⭐ NOAEL의 본거지 (90일·sub-chronic)
3.3.6 Mutagenicity / Genotoxicity 양성이면 사실상 사용 불가 — 결정적 게이트
3.3.7 Carcinogenicity
3.3.8 Reproductive toxicity
3.3.9 Toxicokinetics
3.3.10 Photo-induced toxicity UV filter, 향료에서 중요
3.3.11 Human data HRIPT, 임상
3.3.12 Special investigations
3.3.13 Safety evaluation (incl. calculation of MoS) ⭐⭐⭐ NOAEL + SED + MoS가 한 표에. SCCS의 모범 계산이 그대로 박혀있음.
3.3.14 Discussion
4. Conclusion ⭐⭐⭐ SCCS의 최종 의견 한 문장. 우리 Tier 4 결론이 모방해야 할 화법.
5. Minority opinion
6. References 독성 시험 원문 추적 단서

SCCS Notes of Guidance 12판 §3-2/§3-3에 Opinion 구조와 평가 흐름이 정리됨. 실제 Opinion 예: SCCS Opinions index (PDF 다수 공개).

전략적 통찰 §3.3.13 "Safety evaluation"은 한 표에 NOAEL, SED, MoS가 SCCS의 가정값과 함께 나란히 적혀있다. 단일 섹션 추출만으로 NOAEL + SCCS가 사용한 SED 가정 + 결과 MoS 세 가지를 모두 ground truth로 확보할 수 있다. 이 표를 정확히 파싱하는 것이 Tier 3의 highest-value target.

3. 추출 schema — 구조화 출력 강제

Tier 3의 단일 호출이 반환해야 하는 구조. LLM이 자유 텍스트로 NOAEL을 쓰게 두면 안 된다. schema에 anchor 필드를 강제:

from pydantic import BaseModel, Field
from typing import Literal, Optional

class Citation(BaseModel):
    """모든 추출 사실에 부착되는 출처 anchor."""
    document_id: str = Field(..., description="SCCS Opinion ID, e.g., SCCS/1631/21")
    section: str = Field(..., description="e.g., '3.3.5' or '3.3.13'")
    page: int
    quote: str = Field(..., description="original PDF text, literal — must match exactly")

class NoaelRecord(BaseModel):
    """한 시험에서 추출된 NOAEL. 한 성분에 여러 개 있을 수 있음."""
    value_mg_per_kg_day: float
    species: Literal["rat", "mouse", "rabbit", "dog", "human", "other"]
    species_other: Optional[str] = None
    study_type: Literal[
        "acute", "sub_acute_28d", "sub_chronic_90d",
        "chronic", "reproductive", "developmental", "other"
    ]
    route: Literal["oral", "dermal", "inhalation", "other"]
    oecd_test_guideline: Optional[str] = None  # e.g. "OECD TG 408"
    citation: Citation

class DermalAbsorptionRecord(BaseModel):
    value_pct: float  # SCCS가 사용한 DA% — Tier 1로 feed
    measurement_method: Literal["in_vitro_oecd428", "default", "other"]
    citation: Citation

class SccsSafetyEvaluation(BaseModel):
    """§3.3.13 표를 그대로 옮긴 구조 — SCCS의 worked example."""
    noael_used_mg_per_kg_day: float
    sed_used_mg_per_kg_day: float
    mos_computed: float
    sccs_assumptions: dict  # {amount_g_day, conc_pct, F, DA_pct, BW_kg}
    citation: Citation

class SccsOpinionFinalConclusion(BaseModel):
    """§4 — SCCS의 최종 의견. Tier 4가 우리 B.1 결론 작성 시 참조."""
    verdict: Literal["safe", "safe_with_conditions", "not_safe", "insufficient_data"]
    max_concentration_pct: Optional[float] = None
    use_restrictions: list[str] = []
    citation: Citation

class SccsOpinionExtraction(BaseModel):
    inci: str
    cas: Optional[str] = None
    opinion_id: str
    opinion_date: str  # YYYY-MM-DD
    noaels: list[NoaelRecord]
    dermal_absorption: Optional[DermalAbsorptionRecord] = None
    sccs_safety_evaluation: Optional[SccsSafetyEvaluation] = None
    final_conclusion: SccsOpinionFinalConclusion

이 schema의 두 가지 핵심:

  • Citation 강제: 모든 numeric value 옆에 quote가 박혀있어야 함. 후처리 evaluator가 quote ∈ source PDF text를 literal match 검사.
  • Literal type 강제: species·route·study_type 같은 핵심 카테고리는 free string 금지. LLM이 "rats (Wistar)" 같이 쓰면 enum 위반 → 재시도.

4. Extraction prompt — 핵심 4규칙

System prompt에 박는 4가지. (사용자가 prompt engineering 능숙하므로 패턴만):

  1. "Cite or refuse" — quote 필드가 비어있으면 그 record 자체를 출력하지 마라. 빈 값을 만들어 채우는 것보다 누락이 낫다.
  2. "Choose conservative" — 한 성분에 여러 NOAEL이 있으면 모두 추출한 뒤 별도 함수가 가장 낮은 값을 고른다. LLM에게 "고르라"고 시키지 마라.
  3. "Species and route always" — species나 route가 PDF에 명시 안 되면 그 record를 버려라. 추정 금지.
  4. "Units in your output are mg/kg bw/day" — 원문에 다른 단위면 변환 후 단위 변환 그 자체를 quote에 명시("converted from 5 μg/kg/day = 0.005 mg/kg/day").
Two-pass extraction Pass 1: 섹션별로 chunk → 각 chunk에서 raw record 추출. Pass 2: 같은 성분의 모든 raw record를 LLM에게 다시 보여주며 "가장 보수적 NOAEL과 그 이유를 골라라" — 이 단계가 reasoning이라 Tier 4지만, 결정 자체는 deterministic rule(min)로 검증 가능. LLM은 "왜 그게 가장 보수적인가" narrative만 만든다.

5. 일반 RAG와 다른 점 — Evaluator pattern

추출 후 즉시 별도 evaluator agent를 띄운다. 이게 fabrication을 잡는 결정타:

def verify_extraction(
    extraction: SccsOpinionExtraction,
    source_pdf_text: str
) -> list[Violation]:
    violations = []
    for rec in extraction.noaels:
        # 규칙 1: quote가 원문에 실제로 있는가
        if rec.citation.quote not in source_pdf_text:
            violations.append(Violation("quote_not_in_source", rec))
        # 규칙 2: 출력된 값이 quote에 실제로 등장하는가
        if str(rec.value_mg_per_kg_day) not in rec.citation.quote and \
           str(int(rec.value_mg_per_kg_day)) not in rec.citation.quote:
            violations.append(Violation("value_not_in_quote", rec))
        # 규칙 3: species 단어가 quote 또는 인접 문맥에 있는가
        if rec.species not in rec.citation.quote.lower():
            violations.append(Violation("species_unverified", rec))
    return violations

violations가 비어있지 않으면: (a) 재시도 또는 (b) HITL 트리거. 임계는 운영하며 조정.

단위 변환 함정 PDF에서 "5 mg/kg/day"가 그대로 추출되면 quote에 "5"가 있어 검증 통과. 하지만 원문이 "5 μg/kg/day"였다면? evaluator가 단위 토큰 자체를 quote에 강제 포함시키는 추가 룰이 필요. 모든 numeric record의 quote에 단위 토큰이 literal로 들어가지 않으면 reject.

6. PDF 처리 파이프라인 — 코퍼스 구축

SCCS Opinion은 PDF로만 배포된다. 자동화의 1차 작업:

단계도구·전략주의
1. 수집SCCS Opinions index 크롤링 → 모든 PDF + 메타데이터(opinion_id, 성분, 발행일)EU 공공문서 → 라이선스는 일반적으로 자유. 단, 사내 한정 사용을 명시.
2. 파싱네이티브 텍스트 추출 시도 (pdfplumber/pymupdf) → OCR fallback (Tesseract, AWS Textract). 표는 별도 추출(camelot 또는 LLM 시각 추출).구버전 Opinion은 스캔 PDF — OCR 필수. 표 인식이 §3.3.13 추출의 success/fail를 가른다.
3. 섹션 분할위 anatomy의 번호 패턴(^3\.3\.5\s+Repeated dose)으로 정규식 분할 → 섹션별 store.일부 Opinion은 번호 폰트가 다름·OCR 오류 → 정규식 다중 패턴 + LLM fallback.
4. 청크·임베딩섹션 단위 청크 (작게는 paragraph) → 임베딩. 메타데이터: opinion_id, section_id, page.화학명 임베딩 품질이 일반 임베딩 모델에서 약함 — 화학·생물 도메인 모델 고려 (BiomedBERT, MatSciBERT 등).
5. 검색 전략일반 vector search 단독 X. "성분 + 섹션 번호"로 필터링 후 벡터 검색. NOAEL 찾을 때 §3.3.5만 검색.섹션 인식이 RAG 정확도 1순위 lever.
6. 캐싱한 번 추출한 성분의 결과는 캐시. SCCS Opinion은 갱신 빈도 낮음 (Opinion 자체가 immutable).제품 라인 전체에서 같은 성분 반복 평가 — 캐시가 비용·시간 절반 이상 절감.

7. CIR·ECHA·EFSA로의 확장

SCCS Opinion이 없는 성분은 CIR(Cosmetic Ingredient Review) 또는 ECHA REACH dossier로 fallback. 각자 다른 구조:

  • CIR Final Report: SCCS와 유사한 구조지만 섹션 번호 체계 다름. "Summary of Safety Assessment" 표가 NOAEL의 메인. 미국 단위(주로 mg/kg/day) 사용.
  • ECHA REACH dossier: 화학물질 관점. cosmetic context가 약하지만 NOAEL·LD50 데이터는 풍부. "Endpoint summary" 섹션 중점.
  • EFSA Scientific Opinion: 식품 첨가물·향료. 경구 NOAEL이 풍부. 경구 NOAEL을 dermal 평가에 쓸 때는 흡수율 보정 필요 (SCCS NoG 12판이 명시).

스키마는 source별 별도 클래스 + 통합 view. SCCS를 최우선 신뢰, 다음 CIR, 다음 ECHA, 그 다음 EFSA 순서로 fallback chain을 명시적으로 구성.

8. 사용자 처방으로 가는 통합 흐름

처방의 한 성분이 들어왔을 때 Tier 3가 거쳐가는 의사결정:

# Tier 3 orchestration (pseudocode)

def get_noael_for_substance(inci: str) -> NoaelDecision:
    # 1. 캐시 확인
    cached = cache.get(inci)
    if cached and not cached.stale: return cached

    # 2. SCCS Opinion 우선
    sccs_opinion = sccs_corpus.find(inci)
    if sccs_opinion:
        extracted = extract_with_llm(sccs_opinion, schema=SccsOpinionExtraction)
        violations = verify_extraction(extracted, sccs_opinion.text)
        if not violations:
            chosen = pick_most_conservative(extracted.noaels)
            return NoaelDecision(chosen, source="SCCS", confidence="high")

    # 3. CIR fallback
    cir = cir_corpus.find(inci)
    if cir:
        # 동일 패턴, CIR 스키마로
        ...

    # 4. 모두 실패 → Tier 5 HITL
    return NoaelDecision(None, source="hitl_required", confidence="none",
                         hitl_payload="No SCCS/CIR/ECHA/EFSA NOAEL found for " + inci)

이 함수의 출력이 Tier 1 compute_mos의 noael_mg_per_kg_day 입력이 된다. 모든 NOAEL은 출처와 confidence 등급을 들고 다닌다. Tier 4 reasoning이 "confidence: high (SCCS Opinion §3.3.13)"를 narrative에 그대로 박는다 — SA 검토 시 즉시 검증 가능.

9. Quiz — 4문항

Q1. SCCS Opinion에서 한 성분에 다음 NOAEL 후보가 나왔다: (A) rat 90-day oral 100 mg/kg/day, (B) dog 28-day oral 50 mg/kg/day, (C) rabbit dermal 200 mg/kg/day. MoS 계산에 어느 것을 쓰는가? 이유?

답 보기
가장 보수적(=낮은) 값 = (B) 50 mg/kg/day. 그러나 study_type이 28일(sub-acute) — 정상 평가에서 sub-chronic(90일) 데이터가 선호된다. 따라서 실제 의사결정은 두 단계:
① 모든 record를 추출 (LLM은 고르지 않음)
② deterministic rule: "study quality" 가중치 (chronic > sub-chronic > sub-acute > acute) × "보수성" → 점수화. 단순 min이 아니다. 시스템에 study_type weighting table이 별도 있어야 한다.
답: 보수성만 보면 (B), 데이터 품질 가중치까지 보면 (A)가 baseline + (B)를 cross-check로 명시. 시스템은 두 가지 모두 narrative에 노출.

Q2. LLM이 NOAEL = 75 mg/kg/day를 출력하면서 quote는 "The Panel concluded a NOAEL of 75 mg/kg bw/day"라고 채웠다. 그런데 원문 PDF에는 "75 mg/kg bw/d"로만 쓰여있다 (bw/d vs bw/day). evaluator가 reject하는가? 어떻게 다룰까?

답 보기
Reject 한다 — naive literal match면. SCCS PDF는 "bw/d" "bw/day" "body weight/day" 등 표기 변종이 다수. 따라서 evaluator는 두 가지 중 하나:
① 관대 normalization: quote를 비교 전에 단위 동의어 사전(bw/d ↔ bw/day) 적용. 단, 동의어 사전이 정확해야 함.
② "인근 ±N 토큰" match: literal match는 강제하되 ±N 토큰 거리 허용.
추천: ②와 함께 quote 안에 unit 토큰을 반드시 포함 (mg/kg, kg/d 등 단위 부품). 단위 자체가 quote에 있으면 동의어 변환 위험이 적다. 부분 일치(70%+) 시 통과, 미달 시 HITL.

Q3. 성분 INCI가 "Niacinamide"인데 SCCS Opinion에는 발견되지 않는다 (실제로 niacinamide는 SCCS Opinion이 없는 흔한 성분이다 — 비타민 B3로 EFSA·CIR 데이터가 풍부). 시스템은 어떻게 행동해야 하는가? 그리고 Tier 4 reasoning은 이 사실을 어떻게 노출해야 하는가?

답 보기
Tier 3는 fallback chain을 따라간다: SCCS → CIR → ECHA → EFSA. niacinamide는 EFSA에서 비타민으로 ADI 데이터 가져온 뒤 confidence를 "medium (EFSA-derived, dietary context)"로 기록. Tier 4 reasoning은 narrative에 명시적으로 표기: "NOAEL은 SCCS Opinion 부재로 EFSA Scientific Opinion(reference link)에서 도출. 경구 NOAEL을 dermal exposure 평가에 적용 시 SCCS NoG 12판 §3-6.X에 따라 흡수율 보정 적용했음을 명시." → 이 narrative가 SA 검토를 통과시킨다. 침묵의 fallback이 acceptance를 가장 많이 깎는다 (Lesson 0002 Quiz 4와 연결).

Q4. 코퍼스 구축 시 §3.3.13 표가 OCR로 깨져 NOAEL 컬럼과 SED 컬럼이 한 줄로 합쳐졌다. LLM extraction이 NOAEL/SED를 혼동할 위험이 크다. 두 가지 방어 전략을 제시하라.

방어 1 (이중 추출 + 교차검증): 같은 §3.3.13을 (a) 표 인식 모드(LLM에게 "이건 표다, 컬럼 인식해라" prompt), (b) §3.3.13 본문 산문 추출(SCCS는 표 외에 본문에도 "Resulting MoS is X"를 쓴다)로 두 번 추출 → 값 일치하면 통과, 불일치하면 reject.
