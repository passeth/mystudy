TIER 1Lesson 0003 · 약 35분

노출량·SED·MoS의 결정론적 핵심

LLM이 절대 손대면 안 되는 수학. unit test로 100% 박제하고, 모든 후속 추론의 발판이 되는 영역.

이 레슨의 약속 마치면 (1) SED·MoS 공식과 모든 변수를 외워서 화이트보드에 그릴 수 있고, (2) leave-on face cream 처방의 niacinamide MoS를 실제로 계산해 100 임계와 비교할 수 있으며, (3) 시스템에 박을 함수 시그니처와 unit test 묶음을 설계할 수 있다. ToxTool이 자동화하는 그 계산을 직접 처음부터 다시 구현한다.

1. 마스터 공식 두 개

SED  =  A × C × F × DAp  /  BW
MoS  =  NOAEL / SED  ≥  100
기호의미단위비고
SEDSystemic Exposure Dosagemg/kg bw/day각 성분 단위 일일 전신 노출량
A일일 제품 적용량 (RfE)g/daySCCS NoG 12판 Table 1의 카테고리 기본값
C처방 내 성분 농도g/g (= % ÷ 100)%를 그대로 쓰면 100배 오류
FRetention factor무차원Leave-on = 1.0, Rinse-off ≪ 1
DApDermal absorptionfraction (0~1)측정값 없으면 default 50% = 0.5 (SCCS NoG 12 §3-6.3)
BWBody weightkg성인 default 60 kg (SCCS NoG 12)
NOAELNo Observed Adverse Effect Levelmg/kg bw/day주로 90일 경구 반복투여 시험. Tier 3에서 추출.
MoSMargin of Safety무차원임계 ≥ 100 (= 종간차 10 × 개체차 10)

출처: SCCS Notes of Guidance 12th Revision (SCCS/1647/22, 2023), Chapter 3-6. 실제 시스템에 박을 때는 원문 PDF에서 본인이 한 번 더 확인. 본 레슨은 학습용.

2. 단위 함정 — 시스템 무너뜨리는 0순위 위험

Tier 1은 "쉬워서" 무시하기 쉽지만 실패 모드 90%가 단위에서 나온다.

단위 5종 세트
  • % vs fraction: 4% niacinamide = 0.04 g/g. 4 그대로 쓰면 100배 폭증.
  • g vs mg: A는 g/day, SED는 mg/kg bw/day. 마지막에 ×1000 필요.
  • retention 0 vs 1: rinse-off에 1.0을 적용하면 SED 10~100배 폭증, MoS 100배 폭감.
  • DA% vs DA fraction: 50%로 쓸지 0.5로 쓸지 일관되게.
  • NOAEL 종 차이: rat NOAEL을 ×UF=1로 인간 적용 금지. 종간차 10이 default UF에 이미 들어가 있음 (그래서 MoS ≥ 100).

3. SCCS NoG의 RfE 카테고리 (leave-on 위주)

아래 값은 SCCS NoG 12판이 정의하는 표준 적용량. 시스템 구축 시 원문 Table에서 본인이 추출·검증해 입력하라. 본 레슨에는 학습용으로 일부만:

4. Worked Example — Niacinamide 4% in face cream

설정

자사 leave-on face cream에 Niacinamide(나이아신아마이드, INCI: Niacinamide) 4% 처방. 성인 daily 사용 가정.

Step 1 — 입력값 정리
A = 1.54 g/day (face cream day, SCCS Table)
C = 0.04 (4% → fraction)
F = 1.0 (leave-on)
DAp = ? — niacinamide의 dermal absorption은 측정값이 있는가? SCCS Opinion 또는 CIR 확인. 측정값 없으면 default 0.5.
BW = 60 kg
Step 2 — SED 계산 (default DA 가정)
SED = 1.54 × 0.04 × 1.0 × 0.5 / 60
       = 0.0308 / 60
       ≈ 0.000513 g/kg bw/day = 0.513 mg/kg bw/day
Step 3 — NOAEL 조회 (Tier 3 호출)
Tier 3 RAG에게 "Niacinamide NOAEL repeated dose"로 질의 → CIR 또는 EFSA 데이터. Niacinamide는 비타민 B3로 EFSA에서 ADI 가까운 데이터 있음.
가정: NOAEL ≈ 900 mg/kg bw/day (학습용 예시값, 실제는 RAG가 출처 anchor와 함께 제공해야 함).
Step 4 — MoS 계산
MoS = 900 / 0.513 ≈ 1755.
1755 ≫ 100 → 안전 (큰 여유). 4% 농도는 충분히 보수적.
Step 5 — Aggregate exposure 점검
같은 사용자가 niacinamide 함유 토너+세럼+크림+자외선차단제를 같이 쓸 가능성? SCCS NoG 12판은 aggregate exposure 평가 권장. 합산 SED ≪ NOAEL 인지 별도 확인. 단일 제품 MoS가 1755여도 aggregate에서 100 미달이면 안전하지 않다.
왜 LLM 금지인가 위 계산은 7~10번의 곱·나누기다. 일반 LLM은 다단계 산술에서 무작위 오류를 낸다. 더 심각한 건 오류를 정성적 narrative로 그럴듯하게 가린다는 점. "MoS 175 (safe)" — 한 자리 빠뜨려도 자기검증 못함. SED·MoS·aggregate는 100% deterministic 함수, unit test로 SCCS NoG 예시를 박제.

5. 구현 — 함수 시그니처와 unit test 묶음

# Tier 1 core — pure functions, no LLM

from dataclasses import dataclass

@dataclass(frozen=True)
class Ingredient:
    inci: str
    concentration_pct: float  # 4% → 4.0 (not 0.04)
    dermal_absorption_pct: float | None  # None → use default

@dataclass(frozen=True)
class ExposureInputs:
    daily_amount_g: float       # A (SCCS RfE)
    retention_factor: float     # F: 1.0 leave-on, <1 rinse-off
    body_weight_kg: float = 60.0

DEFAULT_DA_PCT = 50.0  # SCCS NoG 12 §3-6.3
MOS_THRESHOLD = 100.0

def compute_sed(ing: Ingredient, exp: ExposureInputs) -> float:
    """Returns SED in mg/kg bw/day. Pure, deterministic, unit-tested."""
    da = ing.dermal_absorption_pct if ing.dermal_absorption_pct is not None else DEFAULT_DA_PCT
    sed_g = (exp.daily_amount_g
             * (ing.concentration_pct / 100.0)
             * exp.retention_factor
             * (da / 100.0)
             / exp.body_weight_kg)
    return sed_g * 1000.0  # g/kg → mg/kg

def compute_mos(noael_mg_per_kg_day: float, sed_mg_per_kg_day: float) -> float:
    if sed_mg_per_kg_day <= 0:
        raise ValueError("SED must be positive")
    return noael_mg_per_kg_day / sed_mg_per_kg_day

def is_safe(mos: float, additional_uf: float = 1.0) -> bool:
    return mos >= MOS_THRESHOLD * additional_uf  # 데이터 갭 시 UF 상향

핵심 테스트 묶음 (이 다섯 개가 통과 안 하면 시스템 출하 금지):

def test_niacinamide_facecream_baseline():
    # 위 worked example을 박제
    ing = Ingredient("Niacinamide", 4.0, dermal_absorption_pct=None)
    exp = ExposureInputs(daily_amount_g=1.54, retention_factor=1.0)
    sed = compute_sed(ing, exp)
    assert 0.51 < sed < 0.52  # mg/kg bw/day

def test_rinseoff_retention_reduces_sed_100x():
    base = ExposureInputs(1.0, retention_factor=1.0)
    rinse = ExposureInputs(1.0, retention_factor=0.01)
    ing = Ingredient("X", 1.0, 100.0)
    assert compute_sed(ing, base) == 100 * compute_sed(ing, rinse)

def test_percent_not_fraction_input():
    # 4 (% 의미) vs 0.04 입력 혼동 방지 — 4를 넣었을 때 결과가 "그럴듯하게 큰" 값으로 가는지 명시 검증
    ing = Ingredient("X", concentration_pct=4.0, dermal_absorption_pct=50.0)
    exp = ExposureInputs(1.54, 1.0)
    assert 0.5 < compute_sed(ing, exp) < 0.6  # 0.04를 잘못 넣었으면 1/100 작은 값 나옴 → 즉시 실패

def test_mos_threshold_exact_100():
    assert is_safe(100.0) is True
    assert is_safe(99.9) is False

def test_additional_uf_raises_threshold():
    # 데이터 갭(예: 단기 시험만 있음)에서 UF=3 추가 → 임계 300
    assert is_safe(300.0, additional_uf=3.0) is True
    assert is_safe(299.0, additional_uf=3.0) is False
SCCS 예시값을 박제하라 SCCS NoG 12판 본문에는 worked example이 여럿 들어있다. 그 전부를 unit test로 박아두면 향후 코드 리팩토링·라이브러리 교체에도 SCCS 정합성이 깨지지 않는다. 이게 "95% acceptance"의 출발선.

6. Aggregate exposure — Tier 1의 두 번째 층

한 사용자가 동일 성분(예: phenoxyethanol 보존제)을 함유한 여러 제품(클렌저+토너+세럼+크림+자외선차단제)을 동시 사용. SCCS NoG 12판이 강조하는 평가:

SEDaggregate  =  Σ SEDi   over all products containing the substance

구현은 사실상 같은 함수의 합산이다. 다만 시스템 설계 시 입력 모델이 "단일 제품" 모드와 "사용자 시나리오(여러 제품 묶음)" 모드를 모두 받아야 한다. SA는 양쪽 결과를 모두 본다.

7. Quiz — 계산 손에 익히기 (4문항)

Q1. Face cream에 salicylic acid 1.5%. dermal absorption은 측정값 없음(default 적용). SED는?

답 보기
SED = 1.54 × (1.5/100) × 1.0 × (50/100) / 60 = 0.000193 g/kg/day = 0.193 mg/kg bw/day.
체크: 농도가 niacinamide(4%) 대비 ~1/2.7 → SED도 ~1/2.7 (0.513 → 0.193). 단위 일관.

Q2. 같은 salicylic acid의 NOAEL이 SCCS Opinion에서 50 mg/kg bw/day로 추출됐다. MoS는? 안전한가?

답 보기
MoS = 50 / 0.193 ≈ 259. 259 > 100 → 안전 (default UF 기준). 단, salicylic acid는 SCCS Opinion에 leave-on 농도 한도가 별도 존재하는 성분이므로 (예: 0.5% 또는 라벨 경고 조건부) — MoS 외 농도 한도 규제도 별도 검증해야 한다. Tier 1 결과만으로 안전 결론 X.

Q3. 같은 사용자가 niacinamide 4% 함유 토너(A=7.5 g/day, F=1.0), 세럼(A=1.4, F=1.0), 크림(A=1.54, F=1.0)을 모두 매일 사용. niacinamide aggregate SED는?

답 보기
각 제품 SED (DA=50% default):
토너: 7.5 × 0.04 × 1.0 × 0.5 / 60 = 0.0025 g/kg/day = 2.5 mg/kg/day
세럼: 1.4 × 0.04 × 1.0 × 0.5 / 60 = 0.000467 g/kg/day = 0.467 mg/kg/day
크림: 1.54 × 0.04 × 1.0 × 0.5 / 60 = 0.000513 g/kg/day = 0.513 mg/kg/day
Aggregate SED ≈ 3.48 mg/kg bw/day. 단일 크림 SED(0.513)의 약 6.8배. MoS = 900 / 3.48 ≈ 259. 여전히 100 초과지만 단일 제품 1755 대비 7배 작아짐. aggregate 평가가 왜 결정적인지의 모범 예.

Q4. 시스템 구현 시 누군가 "농도를 % 그대로(예: 4) 받자"고 제안한다. 함수 시그니처를 어떻게 설계해 % 오용을 막을 것인가? 한 가지 방어 + 한 가지 fail-fast 패턴.

답 보기
방어 1 (타입 분리): Percentage 또는 Fraction 같은 newtype/branded type을 만들고 concentration_pct: Percentage처럼 표시. mypy/Pyright에서 float 직접 대입 거부.
fail-fast 1 (런타임 검증): __post_init__에서 0 ≤ concentration_pct ≤ 100 강제. 추가로 합계 검증: 한 처방의 모든 ingredient.concentration_pct 합 ≈ 100 (±0.5%) 검증, 미달 시 raise. 처방 합이 안 맞는데 SED 계산이 통과하는 가장 흔한 silent failure를 차단.
극단적이지만 가치 있음: SCCS NoG worked example의 결과값과 매칭하는 acceptance test를 CI에 박제. 단위·공식 어디든 깨지면 즉시 빨갛게.
